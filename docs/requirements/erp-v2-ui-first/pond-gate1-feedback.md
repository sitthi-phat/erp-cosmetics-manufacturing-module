## สิ่งที่ต้องเพิ่ม/แก้ไข (รายหน้า)

### 1. หน้าหลัก (Home)
- มีชื่อ user แสดงซ้ำ 2 ที่ — แก้ให้เหลือจุดเดียว

### 2. Dashboard
- ออกแบบ **dashboard แยกของแต่ละแผนก** มาให้ดู (ไม่เหมือนกัน) — Sale, Stock, Production, QC, Shipping, Finance, Admin
- ถ้า user มีหลาย role → แสดงหลายแผนกให้เลือกสลับได้

### 3. หน้าลูกค้า (Customers)
- **เพิ่ม contact ของลูกค้าได้ไม่จำกัด** (หลายคนต่อลูกค้า)
- **สถานะลูกค้า (Customer Status)** — ทำ mock ของแต่ละสถานะให้ดู:
  - `Lead`: ได้แค่ contact มา ข้อมูลไม่ครบ ยังไม่มีการสั่งซื้อ
  - `Active`: มีการสั่งซื้อสม่ำเสมอ — **config ต่อลูกค้าได้** ว่านับ "สม่ำเสมอ" ที่กี่เดือน
    (option: 1, 3, 6, 8 เดือน — **default 3 เดือน**) ถ้ามี order เข้ามาในรอบเวลา → Active
  - `Inactive`: เคยสั่งซื้อ แต่ไม่ได้ซื้อตามรอบเวลาที่กำหนด (auto จาก config ข้างบน)
  - `Disabled`: ถูก user กด disable เอง (manual)
  - `Blacklist`: เหมือน Disabled แต่สื่อว่าลูกค้ามีปัญหา
- ทุกลูกค้าต้องมี **comment/note** เพื่อบันทึกว่าจัดการอะไรไว้ (เห็นประวัติการจัดการ)
- แสดงว่า**ลูกค้าคนนี้ sale คนไหนดูแล** + **reassign ได้โดย Sale Manager / System Admin**
  (⚠ มี role ใหม่: Sale Manager)
- ข้อมูลเชิงสถานะลูกค้าพวกนี้ต้องไปโชว์ใน **Sale Dashboard**
- เห็น**ประวัติ PO ทั้งหมดของลูกค้าแต่ละราย** + sale ต้อง **search ลูกค้าด้วยเลข PO หรือวันที่สั่งซื้อ** ได้

### 4. หน้า PO
- **หน้าสรุปรายการ PO แต่ละใบ**: ราคา, VAT และ **sale ที่ดูแล**
- **หน้าเพิ่มสินค้า (ตอนสร้าง PO)**:
  - ระบบ **suggest** ว่าลูกค้าต้องการเลือกอะไร → แสดงว่าสินค้านั้นต้องใช้วัตถุดิบอะไรบ้าง มีครบไหม ขาดอะไร
  - **ถ้าวัตถุดิบขาด → เปิด PO ไม่ได้** และให้**ส่งคำขอสั่งซื้อ (Purchase Request) ไปที่ Stock**
  - ใบ Purchase Request ต้องไปแสดงใน **dashboard ของฝ่ายผลิต** ด้วย
- เลือกขายได้ทั้ง **สินค้า BOM** และ **วัตถุดิบตรงๆ** (ขายวัตถุดิบได้ด้วย)
- ราคา default มาจาก **ราคาขายใน BOM** แต่**แก้ไขได้**

### 5. หน้าสต็อกวัตถุดิบ (Stock)
- **เพิ่มวัตถุดิบใหม่ที่ไม่เคยมีใน list ได้** + **กำหนด UOM (หน่วยนับ) ได้** — เพื่อให้รับเข้าของใหม่ถูกต้อง
- ตอนเพิ่มวัตถุดิบใหม่: **config เกณฑ์ "ใกล้หมด" ต่อวัตถุดิบ** — เลือกได้ว่าเป็น **% หรือจำนวน**
  และสถานะใกล้หมดต้องแสดงใน **Stock Dashboard**
- ตอนรับเข้า: กำหนด **ราคาซื้อ และ ราคาขาย แยกกัน** — ราคาขายใช้เป็น default ใน PO

### 6. หน้า BOM
- ต้องมีหน้า**สร้างสูตรการผลิตใหม่** เลือกจากวัตถุดิบ — **ไม่จำเป็นต้องมี stock** แต่ต้องโชว์ข้อมูล stock ของวัตถุดิบประกอบ
- กำหนด**ราคาทุน**ได้ — default คำนวณจากราคารับซื้อของวัตถุดิบ
- กำหนด**ราคาขาย**ได้ — **mandatory**

### 7. หน้าการผลิต (Production)
- รายการสั่งซื้อเข้ามา **เรียง/ดูจากวันจัดส่ง** ได้ + **search จาก PO / ลูกค้า / วันที่จัดส่ง**
- แสดง **potential delay**: เกณฑ์ = 2 วันผลิต + 1 วันส่ง — ถ้าเลยจากนี้ติดสถานะ `Potential Delay` นอกนั้นสถานะปกติ
- สถานะการผลิตให้ครบ: `Received`, `In-Progress`, `Hold`, `Ready to Delivery`, `Delivered`
- **ทุกสถานะ comment ได้** และ**ส่งต่อสถานะไปให้ Sale หรือ Stock** ได้
  (เช่น ผลิตไม่ได้เพราะติดที่ลูกค้า → แจ้ง Sale / ติดที่ stock → แจ้ง Stock)
- **ปรับสถานะได้ตลอด แต่ต้องมี trace กำกับเสมอ** ว่าใครเปลี่ยนจากอะไรเป็นอะไร เมื่อไหร่

### 8. หน้า QC ตรวจรับวัตถุดิบขาเข้า
- **หน้า Supplier Management (SP)** ใหม่:
  - กำหนด **prefix ของเลข lot ต่อ supplier** ได้
  - กำหนดว่า**วัตถุดิบไหนมาจาก supplier ไหน** — วัตถุดิบ 1 ตัวอยู่ได้หลาย supplier
- ตอนรับเข้า: **เลือก supplier** → ระบบ gen เลข lot default ให้ แต่ต้องกรอก**เลขใบรับจาก supplier** ด้วย + **upload เอกสารใบรับ** ได้
- **การคืนของ (Return)**: ของรับมาแล้วตรวจเจอเสียหาย → ทำใบส่งคืนได้ — ระบุเลข lot → ระบบ auto แสดง supplier → แก้จำนวน return → **ตัด stock พร้อม comment บังคับ** (เหตุผลการ adjust stock ที่ไม่มี PO เพราะเป็นการ return)

### 9. หน้าการจัดส่ง (Shipping)
- PO ที่เปลี่ยนสถานะเป็น **Ready to Deliver + QC ผ่าน** จะมาแสดงที่นี่ เลือกจัดส่งได้ 2 แบบ:
  1. เลือกทีละ PO
  2. สร้างใบจัดส่ง 1 ใบ เลือกหลาย PO รวมกัน
  (สุดท้ายได้**ใบจัดส่ง (Delivery Note)** เหมือนกัน)
- สถานะใบจัดส่ง: `Received` (รับจากฝ่ายผลิตแล้ว), `In-Route`, `Delivered`, `Rejected`, `Postponed`
- ใส่ comment ได้ + แก้สถานะได้ตลอด **แต่ต้องมี trace**

### 10. หน้าใบแจ้งหนี้ (Invoice)
- **ออกใบแจ้งหนี้ได้ตลอดเวลา** แต่ต้องแสดง **status ของ PO** ว่าอยู่ stage ไหน
- **แจ้งเตือนลูกหนี้ค้างจ่าย**: ของส่งไปแล้วแต่เงินยังไม่จ่าย — แสดงว่าค้างมานานเท่าไหร่แล้ว

### 11. หน้าสืบย้อนกลับ (Traceability)
- สืบย้อนกลับได้**ทุก module / ทุก activity** เก็บไว้ตลอด
- มี function **archive ลง text file** — สิทธิ์ระดับ **Super User เท่านั้น** (⚠ มี role ใหม่: Super User)

### 12. หน้าตั้งค่าระบบ (Settings)
- ตั้งค่าสิทธิ์เข้าใช้**แต่ละ module ตามเมนูซ้าย** ด้วยสิทธิ์ 5 ระดับ: **Read, Update, Create, Delete, Approve**
- **สร้าง Role ได้ไม่จำกัด** / **User อยู่ใต้ Role**
- แก้ไขที่อยู่บริษัทได้ (company profile)

## ⚠ ประเด็นเชิงระบบที่ต้อง flag
- **Role ใหม่ที่โผล่จาก feedback**: Sale Manager (reassign ลูกค้า), Super User (archive trace)
- **เอกสาร/เอนทิตีใหม่**: Purchase Request (PO→Stock), Delivery Note (รวมหลาย PO), Supplier,
  Customer Contact (หลายรายการ), Customer Note/Comment, เอกสารใบรับจาก supplier (file upload),
  ใบคืนของ (Return)
- **Status journeys ที่ต้องออกแบบให้ต่อเนื่อง** (คำสั่งเน้นของปอนด์): Customer lifecycle,
  PO lifecycle (จนถึง invoice/payment), Production lifecycle (รวม Hold/Delay + cross-notify),
  Shipping lifecycle, Return flow, Purchase Request flow — ทุกการเปลี่ยนสถานะ comment ได้ + trace เสมอ
