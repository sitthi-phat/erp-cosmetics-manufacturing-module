# Gate 1 Feedback จากปอนด์ (รอบที่ 3) — 2026-07-08

ยังไม่ approve — feedback รายหน้า + มี 4 ประเด็นที่สั่งให้ **PO วิเคราะห์โดยเฉพาะ** (ติดป้าย 🔍PO)

## 1. Dashboard
- คำว่า "Dashboard" ปรากฏ 3 จุดในหน้า — ต้องเหลือจุดเดียว — **และให้ตรวจปัญหา label ซ้ำแบบนี้ทุกหน้า**
- 🔍PO: อยากได้ **filter ด้วยช่วงวัน/เดือน** เช่น เลือก "เดือนนี้" → ทุก tile (ลูกค้าประจำ, ห่างหาย,
  PO, ค้างชำระ, ต้องติดตาม) แสดงข้อมูลของเดือนนั้น — ให้ PO คิดว่าทำยังไงให้ **practical
  "Dashboard อยากให้มีคนใช้"** (ระวัง: บาง metric เช่น "ลูกค้าประจำ" นิยามผูกกับ config รอบเวลา
  ของลูกค้าเอง — ต้องตีความว่า filter เดือนหมายถึงอะไรกับ metric แต่ละตัว)

## 2. หน้าลูกค้า
- **Test data ใน mockup ไม่ตรง use case** — กลัว BA/QA มาเล่นแล้วงง ต้องทำ data ให้เล่าเรื่องจริง
  (เช่น ลูกค้า Follow-up ต้องมีเคส Hold ประกอบ, Blacklist ต้องมี comment เหตุผล)
- **"เพิ่มผู้ติดต่อ" ไม่มีหน้าจอ** — ต้องมี
- **หน้า "เพิ่มลูกค้า" กลายเป็นหน้า edit ลูกค้า** — ต้องทำให้ตรง use case จริง (create ≠ edit)

## 3. หน้า PO
- **ไม่เห็น case การแก้ไขสถานะ PO** — ต้องมี UI ให้เห็นว่าเปลี่ยนสถานะ PO ได้ที่ไหน อย่างไร (พร้อม trace)

## 4. หน้าสต็อกวัตถุดิบ (Goods Receipt)
- 🔍PO: **supplier ส่งของหลายวัตถุดิบในใบรับเดียว** ทำยังไง? — ใบรับ (GR) ต้องรองรับ
  **หลายรายการวัตถุดิบต่อ 1 ใบรับ/1 supplier** (multi-line receipt)
- การเลือก supplier หรือวัตถุดิบ → เป็น **dropdown ที่ search ได้** ทุกจุด

## 5. หน้า PR
- อยากได้หน้า **"สร้างคำขอใหม่"** (เต็มจอ ไม่ใช่แค่ปุ่ม)

## 6. หน้า Supplier
- อยากได้หน้า **"เพิ่ม Supplier"** จริง
- หน้าแก้ไข supplier: **ไม่เอา panel ด้านขวาแบบปัจจุบัน** — ข้อมูลวัตถุดิบต้องเลื่อนดูลำบาก —
  **UX/UI คิด layout ใหม่**

## 7. หน้า BOM
- อยากได้หน้า **"สร้างสูตรใหม่"** ที่กดได้จริง — ตอนนี้กดอะไรไม่ได้เลย

## 8. หน้าการผลิต + QC (ประเด็นใหญ่)
- **"QC ไม่ผ่าน" ไม่ควรเป็นตัวเลือกในหน้าการผลิต** — การตัดสิน QC ต้อง manage ที่หน้า QC เท่านั้น
  (หน้าผลิตแค่เห็นผล/รับงานกลับ)
- 🔍PO: **นิยาม QC ราย line item**: PO มีหลาย line item แต่ละตัวมีผล ผ่าน/ไม่ผ่าน —
  ถ้า line item ใดตัวหนึ่งไม่ผ่าน = **PO นั้น QC ไม่ผ่านทั้งใบ แล้วส่งกลับฝ่ายผลิต ใช่ไหม?**
  (ปอนด์ถามยืนยัน — PO วิเคราะห์และเสนอ)
- 🔍PO: **เลข Batch สร้างตอนไหน? QC รู้ได้อย่างไรว่า PO ไหนมาจาก Batch ไหน?** —
  ต้องนิยาม lifecycle ของ Batch (สร้างเมื่อเริ่มผลิต? 1 PO = กี่ Batch?) + ความเชื่อมโยง
  PO ↔ Batch ↔ QC ให้ชัด (หัวใจ GMP traceability)

## 9. หน้าการจัดส่ง
- **ไม่มีหน้าจอ "สร้างรอบจัดส่ง" จริง** — ต้องมีทั้ง 2 ทาง: เลือก PO ก่อนแล้วสร้างรอบ /
  สร้างรอบก่อนแล้วค่อยเลือก PO
- **ข้อมูลรอบจัดส่ง**ต้องมี: **คนขับ, เบอร์ติดต่อคนขับ, Route, ขนาด/ประเภทรถ**
  (เก๋ง / motorcycle / กระบะ / 10 ล้อ)

## หลักการที่ย้ำอีกครั้ง
- Mockup = spec ให้ BA/Engineer/QA — ทุกหน้า create/edit ต้องกดได้จริง เห็น flow จริง
- Test data ต้องสมจริงตรง use case ทุกหน้า
