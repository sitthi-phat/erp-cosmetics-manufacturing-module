## 2. Module: Dashboard รายแผนก (×7) — ตื้นเกินไป
- **ไม่มีรายละเอียดของช่วงเวลา (date filter) ว่าทำงานยังไง** — ต้องอธิบายพฤติกรรมจริง: preset ไหนบ้าง, default อะไร, เปลี่ยนช่วงแล้ว tile ไหนถูกกรองด้วยความหมายอะไร (event vs state ตาม status-journeys §12), custom range เลือกยังไง, refresh/auto-refresh สัมพันธ์กับ filter ยังไง
- **ไม่มีมุมมองของแต่ละแผนก (ทั้ง 7) ว่าเห็นอะไรบ้าง + เงื่อนไขเป็นอย่างไร** — ต้องระบุครบต่อแผนก: tile ทุกตัว, นิยาม/สูตรนับของแต่ละ tile, เงื่อนไขการเข้ากลุ่ม, drill-down เห็นคอลัมน์อะไร, กดรายการแล้วไปไหน, ใครมีสิทธิ์เห็น (Read bit), กรณีหลาย role
