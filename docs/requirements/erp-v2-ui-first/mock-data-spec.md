# Mock Data Spec — ESSENCE Hub System (dataset เดียว ร้อยเรื่องถึงกันทุก module)

slug: `erp-v2-ui-first` · เขียนโดย PO · 2026-07-08 · ให้ UX/UI ฝังลง mockups ทุกหน้า (BA/QA เล่นแล้วเห็นเรื่องเดียวกัน ไม่งง)
ผูกกับ `status-journeys.md` (Batch `B-{PO}-{line}-{run}`, QC ราย line + rework loop, Shipment→DN, GR multi-line)

## สรุปภาษาไทย
Dataset ชุดเดียวที่ทุกหน้าจอดึงไปแสดง — ลูกค้า→PO→BOM→Lot วัตถุดิบ→Batch→QC→รอบจัดส่ง/DN→invoice→noti→trace เชื่อมกันหมด. ครอบ **8 use case การผลิต** (Draft, Hold, QC ผ่านหมด, QC ไม่ผ่านราย line ค้าง rework, rework loop เต็มวง run1→run2, ส่งสำเร็จ 1 PO 1 รอบ, หลาย PO ส่งสำเร็จหมด, หลาย PO สำเร็จแค่ 1 (Partially)). ท้ายไฟล์มีตาราง **"หน้าจอไหนแสดงข้อมูลก้อนไหน"**

> รูปแบบเลข: `PO-2026-XXXX` · Batch `B-{PO}-{line}-{run}` · `DN-2026-XXXX` · Shipment `SH-2026-XXXX` · `INV-2026-XXXX` · `PR-2026-XXXX` · `GR-2026-XXXX` · Lot = prefix ต่อ supplier + running

## B. 8 Use Cases (การผลิต — ร้อยเรื่องถึงกัน)

### UC1 — PO สถานะ Draft
- **PO-2026-0031** · ลูกค้า สวยใส (Lead, CUS-0009) · Sale อารดา · สร้าง 09/07 10:00 · **สถานะ: ร่าง (Draft)** · billing: ยังไม่วางบิล
- line 1: เซรั่มวิตามินซี 15% (BOM-01) × 100 @ 220 · เช็ควัตถุดิบ: วิตามินซี พอ, กลีเซอรีน พอ → ไม่มี PR
- ลูกค้ายังเป็น **Lead** (จะเป็น Active เมื่อยืนยัน PO ใบแรก) · ยังไม่มี Batch · trace: "สร้าง PO (ร่าง) 09/07 10:00 · อารดา"

### UC2 — PO เข้าผลิตแล้วถูก Hold
- **PO-2026-0012** · ลูกค้า เดอร์มา แคร์ (Follow-up, CUS-0021) · Sale สมหญิง · ยืนยัน 04/07 · **สถานะ: กำลังผลิต (มี line ติด Hold)**
- line 1: โลชั่นบำรุงผิว (BOM-03) × 200 → Batch **B-PO-2026-0012-1-1** · กำลังผลิต · Lot ใช้: L-GLY-2506, L-OLV-2604
- line 2: โฟมล้างหน้า (BOM-04) × 300 → Batch **B-PO-2026-0012-2-1** · **Hold** (ลูกค้าขอแก้สูตร) · raise Sale
- **cross-module:** Production Hold (line 2) → raise Sale สมหญิง → Sale ตั้งลูกค้า CUS-0021 = **Follow-up** + comment "โทรยืนยันการแก้สูตร PO-2026-0012" → โผล่ tile "ต้องติดตาม" (Sale Dashboard)
- trace: 04/07 09:12 สร้างงานผลิต · 08/07 09:40 วิชัย(ผลิต) Hold line2 raise Sale · 08/07 10:20 สมหญิง ตั้งลูกค้า Follow-up
- noti: → Sale (Hold, deep link po-detail line2) · → Customer detail (Follow-up)

### UC3 — PO ส่ง QC ผ่านทั้งหมด

### UC4 — PO QC ไม่ผ่านราย line (ค้าง rework)
- **PO-2026-0022** · ลูกค้า กลอรี่ (Active, CUS-0025) · Sale สมหญิง · ยืนยัน 06/07 · **สถานะ: กำลังผลิต (line 2 rework)** — ยังไม่พร้อมจัดส่ง

### UC5 — Rework loop เต็มวง (run1 ไม่ผ่าน → run2 ผ่าน → พร้อมส่ง)
- **PO-2026-0015** · ลูกค้า กลอรี่ (Active, CUS-0025) · Sale สมหญิง · ยืนยัน 03/07 · **สถานะ: พร้อมจัดส่ง**
- line 1: โลชั่นบำรุงผิว (BOM-03) × 200
  - **Batch B-PO-2026-0015-1-1 (run 1):** QC **ไม่ผ่าน** 05/07 · feedback "ความหนืดต่ำกว่ามาตรฐาน" · Lot: L-GLY-2506, L-OLV-2604
  - กด "ผลิตซ้ำ" → **Batch B-PO-2026-0015-1-2 (run 2):** ผลิต 06/07 → QC **ผ่าน** 07/07 (อรุณี)
- ทุก line ผ่าน → PO **พร้อมจัดส่ง**
- **แสดงชัด 2 run:** run1 (ไม่ผ่าน+feedback) และ run2 (ผ่าน) พร้อม timeline · คิว QC เคยเห็น run2 เป็นรายการใหม่ + ประวัติ run1
- trace: 05/07 run1 ไม่ผ่าน · 06/07 ผลิตซ้ำ gen run2 · 07/07 run2 ผ่าน → PO พร้อมจัดส่ง
