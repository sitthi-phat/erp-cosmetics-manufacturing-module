ปอนด์ review mockups รอบ 2 แล้ว **ยังไม่ approve** — feedback ละเอียดรายหน้า
คำสั่งกำกับ:
- **หน้าจอชุดนี้จะถูกส่งให้ BA / Engineer / QA — ต้องละเอียด ไม่งั้นเขาจะทำไม่ครบ**
- **PO agent: ไม่เข้าใจ flow หรืออะไรก็ตาม ให้ถามปอนด์ได้เลย**

## 1. หน้าหลัก (Home)
- มีคำว่า "หน้าหลัก" 2 จุด — ให้เหลือที่เดียว

## 2. Dashboard (ทุกแผนก)
- มีปุ่ม **refresh** + **auto-refresh ทุก 15 วินาที เป็น default**
- refresh แล้ว **ต้องค้างอยู่ที่ view เดิม** (เช่น อยู่ที่ ฝ่ายขาย→ห่างหาย ก็ต้องอยู่รายการนั้นต่อ ไม่เด้งกลับหน้าแรก)
- **ทุก KPI tile ต้อง drill-down ได้**:
  - **ฝ่ายขาย**: กด ลูกค้าประจำ / ห่างหาย / PO เดือนนี้ / ค้างชำระ → แสดง**รายชื่อลูกค้ากลุ่มนั้น + pagination**
    - เพิ่ม tile **"ต้องติดตาม"** ใน Sale dashboard (พฤติกรรมเหมือน tile ลูกค้าประจำ)
  - **คลัง**: กด ใกล้หมด / ล็อตรอ QC / คำขอ PR ค้าง → แสดงรายการ + กดรายการ → **เข้า module นั้นพร้อมข้อมูลที่กด**
  - **ฝ่ายผลิต**: กด คิวงานผลิต / เสี่ยงล่าช้า / พักงาน (Hold) / เสร็จวันนี้ → แสดงรายการ + กดเข้าไปจัดการได้
  - **QA/QC, จัดส่ง, การเงิน, Admin**: ทำงานแบบเดียวกันทุกแผนก

## 3. หน้าลูกค้า
- ลูกค้า "**ต้องติดตาม**" ต้อง manage ได้: เปลี่ยนสถานะเป็น "ต้องติดตาม" ได้ พร้อม **comment (free text)** ว่าติดตามด้วยเรื่องอะไร

## 4. หน้า PO
- **Search ด้วย: วันที่สร้าง / วันที่จัดส่งจริง / วันที่ต้องการรับสินค้า**
- ตอนเพิ่มสินค้า (ทั้ง BOM และวัตถุดิบ): user **แก้จำนวน และ ราคา/หน่วย ได้เสมอ** — **ราคา set เป็น 0 ได้**
- **PO cancel ได้ทุก case** — และ PO ที่ cancel แล้ว **เปลี่ยนกลับเป็น Draft และรับงานต่อได้**

## 5. หน้าสต็อกวัตถุดิบ
- ราคาซื้อ และ ราคาขาย **set เป็น 0 ได้**
- ต้องมี**หน้าจอครบทุก flow ของ stock** โดยเฉพาะ **หน้ารับเข้า (Goods Receipt)** เต็มรูปแบบ:
  - ฟิลด์ lot + supplier ที่ต้องกรอก
  - **อ้างอิงเลข PR ได้ (search เลข PR ได้)** — ถ้ารับเข้าโยงกับ PR ให้**ปิดสถานะ PR อัตโนมัติ**
    พร้อมเหตุผลว่ารับจาก lot ไหน

## 6. หน้าคำขอสั่งซื้อ (Purchase Request)
- **สร้าง PR ตรงจากหน้านี้ได้** (ไม่ใช่เกิดจาก PO อย่างเดียว)
- สถานะ PR:
  - "**ของเข้าครบ**" — เปลี่ยนอัตโนมัติจากการทำรับใน module สต็อกวัตถุดิบ
  - "**รับทราบ**" และ "**ปิดคำขอ**" — เปลี่ยน manual ได้

## 7. หน้า Supplier
- **เพิ่ม/แก้ไข supplier** ได้ พร้อม **ผูกวัตถุดิบรายตัว + ราคารับซื้อ (ต่อ supplier ต่อวัตถุดิบ)**
  - ตอนเพิ่มวัตถุดิบให้ **search วัตถุดิบได้** (ผูกง่าย)
- **ไม่มี function "รับวัตถุดิบเข้าคลัง" ในหน้านี้** (แยกไปอยู่ stock)
- **Active/Inactive supplier ได้**

## 8. หน้าสูตรการผลิต (BOM)
- ต้องเห็น**หน้าสร้างสูตรใหม่** (เต็มรูปแบบ)
- **ราคาทุน**: คำนวณจากราคาวัตถุดิบ ณ ตอนนั้น — ถ้าวัตถุดิบมีหลาย supplier ให้ใช้**ราคาสูงสุด**
  โดยนับเฉพาะ **supplier ที่ active เท่านั้น** — แล้ว user **แก้ทับได้**
- เมื่อบันทึกสูตร → **snapshot ราคานี้ไว้ ไม่คำนวณใหม่**

## 9. หน้าการผลิต
- **Flow ที่ปอนด์กำหนด**:
  - Normal: **รับงาน → กำลังผลิต → ตรวจคุณภาพ (QC) → พร้อมส่งมอบ**
  - QC ไม่ผ่าน: รับงาน → กำลังผลิต → QC → **กลับไปกำลังผลิต** (พร้อม feedback)
  - Hold: รับงาน → กำลังผลิต → QC → กำลังผลิต → **Hold → แก้ไข PO → กำลังผลิต → QC → พร้อมส่งมอบ**
    - Hold เปลี่ยนได้ พร้อม **raise ไปที่ Sale**
- **คำวินิจฉัยของปอนด์เรื่อง "ส่งมอบแล้ว"**: ถ้าหมายถึง delivered ของใบจัดส่ง —
  **ไม่ควรอยู่ในหน้าการผลิต** — หน้าผลิตควรจบที่ **"พร้อมส่งมอบ"** แล้วส่งต่อให้การจัดส่ง
  (PO agent: ยืนยัน/ปรับ journey ตามนี้ ถ้ามีข้อแย้งให้ถามปอนด์)

## 10. หน้า QC
- QC ไม่ผ่าน → **กลับไป "กำลังผลิต" พร้อม feedback**

## 11. หน้าการจัดส่ง + หน้าใบจัดส่ง (ต้อง redesign flow ให้หายสับสน)
- **หน้าการจัดส่ง** = ที่**สร้างใบจัดส่ง**: เลือก PO ก่อนแล้วสร้าง หรือสร้างใบเปล่าแล้ว search PO เพิ่ม —
  **เลือกได้เฉพาะ PO สถานะ "พร้อมจัดส่ง" เท่านั้น** แต่ search เห็นได้ทั้งหมด
- **Search PO ด้วย: PO ID หรือข้อมูลลูกค้า** (ชื่อ, นามสกุล, ชื่อบริษัท, เบอร์โทร contact, ชื่อ/นามสกุล contact)
- **หน้าใบจัดส่ง** = ใบจัดส่งราย order ที่ **print ได้ทีละใบ** (ให้ลูกค้าเซ็น)
- **DN ถูกปฏิเสธ/เลื่อน**: order นั้นเปลี่ยนสถานะกลับ "พร้อมจัดส่ง" + **raise ไปที่ Sale**
  - เคส **Postpone**: PO = พร้อมจัดส่ง + มี **flag กันจัดส่ง "Postpone" พร้อมระบุวันที่** —
    ค้างอยู่ในคิวจัดส่งเพื่อให้ฝ่ายจัดส่งรู้ว่ามี PO ที่เลื่อน

## 12. หน้า Traceability
- **Search ได้ทุก entity**: ลูกค้า, ใบสั่งซื้อ (PO), วัตถุดิบ, PR, Supplier, BOM —
  ในมุมมองของ การผลิต (status), การจัดส่ง (ใบจัดส่ง + DN)
- ค้นด้วย **date range + time**: วันที่สร้าง, วันที่ส่ง
- **เลือกประเภทข้อมูลที่จะ trace ได้** (ลูกค้า/PO/วัตถุดิบ/PR/Supplier/BOM/การผลิต/การจัดส่ง)
- ต้องเห็น **audit ระดับ field**: ใครสร้างลูกค้า, ใครเปลี่ยนราคา, ทุกการแก้ไข —
  **ใครทำอะไร ค่าไหนเปลี่ยน จากอะไรเป็นอะไร**

## 13. หน้าตั้งค่าระบบ — ต้องเห็นหน้าจอจริงครบ 5 หน้า
1. **Role & สิทธิ์** รวมถึงการสร้าง role
2. **ผู้ใช้งาน** รวมถึงการสร้าง user
3. **Config VAT + วันที่มีผล (effective date)**
4. **ข้อมูลบริษัท**
5. **Audit log**
