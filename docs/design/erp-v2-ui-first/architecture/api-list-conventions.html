API Spec — มาตรฐานทุกหน้า List (filter/sort/paging/empty)

ทุก list endpoint ใช้ query param ชุดเดียวกัน + ระบุ ตัวกรอง/การเรียง/ค่าเริ่มต้น/ขนาดหน้า/empty state ต่อหน้าให้ครบ (Engineer/QA ไม่ต้องเดา). อ้าง functional-spec/list-conventions (10 หน้า) + envelope มาตรฐาน (ADR-004).

← Architecture Functional spec (BA)

ในหน้านี้

param ร่วม ตัวกรอง/เรียง ต่อหน้า

Query param ร่วม (ทุก list)

?page (เริ่ม 1) · ?pageSize (default 20, เลือก 20/50/100) · ?sort (field:asc|desc) · ?q (ค้นข้อความ) · + ตัวกรองเฉพาะหน้าตามตารางล่าง. response = envelope + meta.total สำหรับ pager. empty state: total=0 → HTTP 200 + list ว่าง + ข้อความมาตรฐาน "ไม่พบรายการ" (+ปุ่มล้างตัวกรองถ้ามี filter). trace/audit ใช้ keyset paging; list module ปกติใช้ offset paging.

ตัวกรอง / การเรียง ต่อหน้า List (10 หน้า)

หน้า / endpointตัวกรอง (filter)เรียงได้ (sort)default sort
customers
GET /api/customers		status (6 ค่า), sale_owner, ค้นด้วยเลข PO, ช่วงวันสั่งซื้อ, q(ชื่อ/บริษัท/เลขภาษี)ชื่อ, สถานะ, วันสร้าง, ยอดสั่งล่าสุดวันสร้าง desc
po-list
GET /api/po		3-date (created/actual_ship/want)+range, fulfilStatus, billingStatus, saleOwner, q(PO/ลูกค้า)เลข PO, วันสร้าง, วันส่ง, ยอด, สถานะวันสร้าง desc
stock
GET /api/stock		near_low(bool), negative(bool), มี/ไม่มี threshold, q(ชื่อ/รหัส)ชื่อ, คงเหลือ, มูลค่า, สถานะชื่อ asc (ใกล้หมด/ติดลบ ปักบน)
supplier
GET /api/suppliers		active/inactive, q(ชื่อ), มีวัตถุดิบ Xชื่อ, จำนวนวัตถุดิบ, สถานะชื่อ asc
purchase-request
GET /api/purchase-requests		status(6), source(auto/manual), ช่วงวันที่, q(PR/วัตถุดิบ)เลข PR, วันสร้าง, สถานะวันสร้าง desc
invoices
GET /api/invoices		status(รอชำระ/ชำระ/เกินกำหนด), ช่วงวันออก/ครบกำหนด, overdue(bool), q(INV/ลูกค้า)เลข INV, วันออก, ครบกำหนด, ยอด, วันค้างวันออก desc
qc
GET /api/qc/queue		target(lot/batch), qc_status, ช่วงวันที่, q(lot/batch/PO)วันเข้าคิว, ชนิด, สถานะวันเข้าคิว asc (เก่าสุดก่อน)
shipping
GET /api/shipping/queue		postpone(bool), q(PO ID/ข้อมูลลูกค้า: ชื่อ/บริษัท/เบอร์/ชื่อ contact)วันต้องการรับ, ลูกค้า, Postponeวันต้องการรับ asc
delivery-note
GET /api/shipping/rounds / dn		รอบ status, DN status(กำลังนำส่ง/ส่งถึง/ปฏิเสธ/เลื่อน), ช่วงวันที่, q(SHP/DN/PO)วันสร้างรอบ, สถานะวันสร้างรอบ desc
bom
GET /api/boms		cost_stale(bool badge ล้าสมัย), q(ชื่อสูตร)ชื่อ, ราคาทุน, ราคาขาย, วันแก้ล่าสุดชื่อ asc
audit log (settings)
GET /api/audit-log		entity_type, actor, action, date-range+time, q(entity id/field)เวลา (keyset)เวลา desc
