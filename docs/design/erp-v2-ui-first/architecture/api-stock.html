API Spec — Stock / Goods Receipt

คลังทำงานบน "บัญชีเดินสะพัด" (append-only ledger) — ยอดคงเหลือคำนวณจากประวัติ ไม่ใช่ตัวเลขเดียว. รองรับสต็อกติดลบ + รับเข้า (GR) หลายบรรทัด + gen lot + ปิด PR อัตโนมัติ + ชดเชยยอดติดลบ. ราคาซื้อ/ขาย 0 ได้.

อ่านสต็อก

GET/api/stock
รายการวัตถุดิบ + 3 ยอด: คงคลัง(on_hand) / จองแล้ว(reserved) / ใช้ได้(available) · สถานะ "ใกล้หมด" (available ≤ threshold) / "ติดลบ" (on_hand<0) / "จองเกิน" (available<0). ค้นได้. Read stock
GET/api/stock/{materialId}/ledger
ประวัติ movement ทั้งหมด (รับ/ตัดผลิต/คืน/ปรับ/retro) — ไล่ที่มาของทุกยอด. keyset paging.
GET/api/stock/dashboard
tile คลัง: ใกล้หมด / ล็อตรอ QC / คำขอ PR ค้าง — drill-down เป็นรายการ + กดเข้า module (auto-refresh 15s).

3 ยอด + availability (reservation layer)

ปอนด์ยืนยัน reservation layer: จองวัตถุดิบตอน PO Confirmed (Reserved เพิ่ม) → ตัดจริง (Consume FIFO) ตอน เริ่มผลิต. ยอด ใช้ได้ (available) = on_hand − reserved ใช้รับปาก PO ใหม่ · จองเกิน (available<0) = เตือนไม่บล็อก. ดู ADR-001 addendum + stock-reservation.md.
GET/api/stock/availability?materialId=
คืน 3 ยอด ต่อวัตถุดิบ: { onHand, reserved, available } (available=onHand−reserved) + flags overReserved(available<0), negative(onHand<0). ใช้ใน stock grid, PO suggest, dashboard tile ใกล้หมด. Read stock
GET/api/stock/{materialId}/reservations
มุมมอง "วัตถุดิบนี้ถูกจองโดย PO ใดบ้าง" (reservation ledger, US-STK-06): list po_line + qty + status(reserved/consumed/released).
reservation ไม่แตะ physical ledger — RESERVE/RELEASE/CONSUME เป็น transition ของตาราง reservation (+cache reserved_balance). ตอนเริ่มผลิต: reserved→consumed พร้อม physical production_consume (−on_hand FIFO, ติดลบได้). ดู api-production › start.

มูลค่าสต็อก (Stock value) — ปอนด์ยืนยัน: ราคาซื้อล่าสุดของ lot

COGS อยู่นอก scope (ADR-001 §4). มูลค่าสต็อกใช้สูตร: ต่อวัตถุดิบ = on_hand × buy_price ของ lot ล่าสุด ของวัตถุดิบนั้น (lot ล่าสุด = lot.received_at มากสุด). รวมทั้งคลัง = Σ ทุกวัตถุดิบ. ยอดติดลบ → มูลค่าติดลบตามจริง (ไม่ clamp). ใช้เป็น tile "มูลค่าสต็อก" ใน dashboard.
GET/api/stock/value
คืนมูลค่ารวม + breakdown ต่อวัตถุดิบ (on_hand, latest lot buy_price, value) เรียงมูลค่าสูงสุด. Read stock
ต้องมี lot.buy_price (บันทึกตอน Goods Receipt line) + index (material_id, received_at desc) เพื่อหา "lot ล่าสุด" เร็ว — ดู db-schema lot.

วัตถุดิบ (master)

POST/api/materials
เพิ่มวัตถุดิบใหม่ + UOM + เกณฑ์ใกล้หมด (ไม่มี default: null=เงียบ; %หรือจำนวน) + ราคาซื้อ/ขาย (0 ได้). Create stock
PATCH/api/materials/{id}
แก้ข้อมูล/เกณฑ์/ราคา (audit ทุก field). ลบ = soft-delete (ADR-009: ของเดิมใช้จนหมด, ห้ามใส่ BOM/PR ใหม่).

รับเข้าคลัง — Goods Receipt (multi-line)

POST/api/goods-receipts
บันทึกรับเข้า 1 ใบ (1 supplier หลายบรรทัด): header (supplier, เลขใบรับ, วันที่, แนบไฟล์) + lines[{material, qty, buy_price, sell_price, pr_id?}]. ระบบทำในทรานแซกชันเดียว:
① gen เลข GR (gapless) · ② gen lot รายบรรทัด (prefix ตาม supplier, สถานะ "รอตรวจรับ") · ③ เขียน movement บวก · ④ retro-alloc ผูก consumption ติดลบเข้า lot ใหม่ (FIFO) + บวกชดเชย · ⑤ ปิด/อัปเดต PR ที่อ้าง (ครบ→"ของเข้าครบ", ไม่ครบ→"รับบางส่วน" + เสนอสร้าง PR ใหม่) · ⑥ noti Stock/Production. Create stock · แนบไฟล์ผ่าน StorageService (ADR-006) หน้า goods-receipt
notice ชดเชยติดลบ: ก่อนยืนยัน ถ้ายอดวัตถุดิบตัวนั้นก่อนรับ < 0 → API คืน compensatesNegative: X ให้ UI ขึ้นกล่อง "การรับนี้ชดเชยยอดติดลบ X หน่วย". นี่คือผลลัพธ์ธรรมชาติของการ net ยอด (ADR-001) ไม่ใช่ logic พิเศษ.
