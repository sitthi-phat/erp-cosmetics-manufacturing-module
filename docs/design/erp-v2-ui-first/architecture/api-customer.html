API Spec — Customer (6 สถานะ)

ลูกค้ามี 6 สถานะ (ผู้สนใจ/ลูกค้าประจำ/ห่างหาย/ต้องติดตาม/ปิดใช้งาน/บัญชีดำ). "ต้องติดตาม" เป็นสถานะแยกจริง ตั้งโดย Sale/Sale Manager พร้อม comment บังคับ. contact ไม่จำกัด, note timeline, reassign, ประวัติ PO, ค้นด้วย PO/วันที่.

ในหน้านี้

อ่าน/ค้น/dashboard สร้าง/แก้ + contact/note เปลี่ยนสถานะ error

อ่าน / ค้นหา / dashboard

GET/api/customers
รายการ + pagination · filter status (6 ค่า) · ค้นด้วยเลข PO หรือวันที่สั่งซื้อ (?q, ?poNo, ?orderFrom&orderTo) · โชว์ sale ที่ดูแล. Read customers
GET/api/customers/{id}
รายละเอียด: contact ทั้งหมด, note timeline, sale owner, ประวัติ PO ทั้งหมด, สถานะ+เหตุผล Follow-up. หน้า customer-detail
GET/api/customers/dashboard
Sale dashboard tiles + drill-down: ลูกค้าประจำ / ห่างหาย / ต้องติดตาม / PO เดือนนี้ / ค้างชำระ — แต่ละ tile คืน list + pagination + deep link เข้า customer detail (auto-refresh 15s, ค้าง view เดิม). Read customers

สร้าง / แก้ + ผู้ติดต่อ / โน้ต

POST/api/customers
สร้างลูกค้า (สถานะเริ่ม "ผู้สนใจ") + ผู้ติดต่อหลัก + เลขผู้เสียภาษี 13 หลัก + credit_days + inactive_after_months (default 3). Create customers หน้า customer-create
PATCH/api/customers/{id}
แก้ข้อมูล (audit ทุก field, ส่ง version กันชน). ลบ = soft-delete (ADR-009: PO เดิมวิ่งต่อ, ห้ามเปิด PO ใหม่).
POST/api/customers/{id}/contacts
เพิ่มผู้ติดต่อ (ไม่จำกัด) · ตั้งผู้ติดต่อหลัก. ลบผู้ติดต่อหลักคนเดียวไม่ได้ (ต้องตั้งใหม่ก่อน). หน้า contact-create
POST/api/customers/{id}/notes
เพิ่มโน้ต timeline (บันทึกการจัดการ).

เปลี่ยนสถานะ (action endpoints)

Endpointทำอะไร (cascade)สิทธิ์ / comment
POST /customers/{id}/follow-up→ต้องติดตาม (จาก Lead/Active/Inactive) · โผล่ tile "ต้องติดตาม" Sale dashboard · ผูก use case Production Hold เหตุลูกค้า (C7→C2b)Sale/Sale Manager · comment บังคับ
POST /customers/{id}/clear-follow-upต้องติดตาม→Active (ถ้ามี order) / InactiveSale · comment optional
POST /customers/{id}/disable→ปิดใช้งาน (ซ่อน/เตือนตอนเปิด PO)Sale Manager/Admin · comment บังคับ
POST /customers/{id}/blacklist→บัญชีดำSale Manager/Admin · comment บังคับ
POST /customers/{id}/unblacklistบัญชีดำ→Active (ปลด)Admin customers · comment บังคับ
POST /customers/{id}/reassignเปลี่ยน sale owner · noti sale เดิม+ใหม่ (C14)Admin/Sale Manager · trace
Lead→Active (PO ใบแรก) และ Active↔Inactive (รอบ config) เป็น auto (cascade/scheduler) ไม่ใช่ endpoint ตรง — badge อธิบายเหตุผล.

error

codeเมื่อไร
CUSTOMER_FOLLOWUP_COMMENT_REQUIREDตั้งต้องติดตามโดยไม่ใส่เหตุผล
CONTACT_PRIMARY_REQUIREDลบผู้ติดต่อหลักคนเดียว
CUSTOMER_HAS_ACTIVE_POลบลูกค้าที่มี PO active (soft-delete: PO เดิมวิ่งต่อ, ห้าม PO ใหม่)
FORBIDDEN_RUCDAAสิทธิ์ไม่พอ (เช่นปลดบัญชีดำต้อง Admin)
