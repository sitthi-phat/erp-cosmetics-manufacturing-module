Role & สิทธิ์ ผู้ใช้ VAT + บริษัท Audit log error

Role & สิทธิ์ (RUCDAA)

GET/api/roles
รายการ role + matrix สิทธิ์ราย module (6 บิต RUCDAA). Read settings
POST/api/roles
สร้าง role (ไม่จำกัด) + ตั้งสิทธิ์ราย module. Admin settings
PATCH/api/roles/{id}/permissions
แก้ matrix สิทธิ์ (audit ทุกช่อง). แก้บิต r = เปลี่ยนผู้รับ noti ของ module นั้นทันที (ADR-005).
DEL/api/roles/{id}
soft-delete: ห้ามลบถ้ายังมี user ผูก (ADR-009 — ต้องย้าย user ก่อน). Admin

ผู้ใช้งาน

POST/api/users
สร้าง user ใต้ role + ตั้ง username/password (bcrypt). Admin settings
POST/api/users/{id}/link-google
ผูกบัญชี Google (Admin กรอก sub/email) → เปิด Google Login (ADR-007).
DEL/api/users/{id}
soft-delete + ปิด login · ต้อง reassign ลูกค้า/งานค้างก่อน (ADR-009). ประวัติการกระทำคงอยู่.

VAT + วันมีผล + ข้อมูลบริษัท

POST/api/vat-configs
ตั้งเรต VAT + effective date (เก็บเป็นประวัติ). invoice ยึดเรตที่มีผล ณ วันออก (snapshot). Admin settings
PATCH/api/company-profile
ข้อมูลผู้ออกใบกำกับภาษี (ชื่อ/ที่อยู่/เลขผู้เสียภาษี/โลโก้) — ใช้ในหน้า invoice-print.

Audit log

GET/api/audit-log
ค้น audit ระดับ field ทั้งระบบ (ใครทำอะไร ค่าไหนเปลี่ยน จากอะไรเป็นอะไร) + date-range/time + entity. keyset paging (1.2M แถว <2s). Read settings (หรือ Trace)
Archive/purge audit เป็น text file = เฉพาะ Super User (ดู api-trace). เก็บ online 1 ปี ไม่มี auto-purge (NFR).

error

codeเมื่อไร
ROLE_HAS_USERSลบ role ที่ยังมี user
USER_HAS_ASSIGNMENTSลบ user ที่ยังมีลูกค้า/งานค้าง (ต้อง reassign)
VAT_EFFECTIVE_OVERLAPวันมีผลซ้อนทับเรตเดิม
