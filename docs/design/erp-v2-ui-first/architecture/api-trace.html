API Spec — Traceability + Audit ระดับ field

ค้นได้ทุก entity (ลูกค้า/PO/วัตถุดิบ/PR/Supplier/BOM/การผลิต/การจัดส่ง) · มุมมองการผลิต (status) + การจัดส่ง (รอบ+DN) · date-range + time · audit ระดับ field (ใครทำอะไร ค่าไหนเปลี่ยน จากอะไรเป็นอะไร) · GMP chain raw Lot→Batch→FG · archive เป็น text file เฉพาะ Super User.

ค้นหา / audit ระดับ field

GET/api/trace/search
?entityType=customer|po|material|pr|supplier|bom|production|shipping · ?q (เลข/ชื่อ) · ?from&to (date-range + time) · ?dateField=created|shipped · includeDeleted. คืนไทม์ไลน์ + audit ระดับ field (timestamp | user | entity | field | old → new). keyset paging (1.2M แถว <2s เพดาน 3s). Read trace หน้า trace
GET/api/trace/{entityType}/{id}
ประวัติเต็มของ entity เดียว: การเปลี่ยนสถานะ + ทุก field ที่แก้ + เหตุผล/comment. อ่านจาก audit_log เดียว (ADR-003).

GMP chain (raw Lot → Batch → FG)

GET/api/trace/gmp/{batchId}
ไล่สาย GMP: Batch → วัตถุดิบ Lot ที่ใช้ (จาก batch_material_consumption) → supplier/GR → PO/ลูกค้า. รวม lot ที่ผูกย้อนหลังด้วย retro-alloc (กรณีเคยตัดสต็อกติดลบ) — คง trace แม้ผลิตก่อนของเข้า (ADR-001).

archive / purge (Super User เท่านั้น)

POST/api/trace/archive
export audit ช่วงวันที่เป็น text file (ผ่าน StorageService → disk/GCS) แล้วอาจ purge แถวที่ export. ทั้ง export และ purge ถูก audit เอง. เก็บ online 1 ปี ไม่มี auto-purge (NFR). สิทธิ์: Super User (Admin trace)

error

codeเมื่อไร
TRACE_ARCHIVE_FORBIDDENไม่ใช่ Super User
TRACE_RANGE_TOO_LARGEช่วงค้นกว้างเกิน (แนะให้ใส่ date-range)
