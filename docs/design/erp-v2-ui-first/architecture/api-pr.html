API Spec — Purchase Request (PR)

PR เกิด 2 ทาง: auto จาก PO วัตถุดิบขาด หรือ Stock สร้างตรงจากหน้า PR. "ของเข้าครบ" = auto จาก Goods Receipt (ผูก lot). "รับทราบ" + "ปิดคำขอ" = manual. รับไม่ครบ → "รับบางส่วน" + เสนอสร้าง PR ใหม่. PR ยกเลิก (void) ไม่ลบ.

ในหน้านี้

อ่าน สร้าง สถานะ error

อ่าน

GET/api/purchase-requests
รายการ + pagination + filter สถานะ (เปิด/รับทราบ/รับบางส่วน/ของเข้าครบ/ปิด/ยกเลิก) + source (auto|manual). โผล่ Stock + Production dashboard. Read stock
GET/api/purchase-requests/{id}
รายละเอียด + วัตถุดิบที่ขอ + PO ต้นทาง (ถ้า auto) + lot ที่รับเข้าปิดคำขอ.

สร้าง (ตรงได้)

POST/api/purchase-requests
Stock สร้างตรง: หลายวัตถุดิบต่อคำขอ (dropdown ค้นได้) + อ้าง PO ได้ · gen PR-{6} (gapless) · สถานะ "เปิดคำขอ". Create stock หน้า pr-create
PR แบบ auto เกิดจาก POST /po/{id}/confirm เมื่อวัตถุดิบขาด (C3/C15) — ไม่ใช่ endpoint นี้.

เปลี่ยนสถานะ

Endpointทำอะไรสิทธิ์

POST /purchase-requests/{id}/closeของเข้าครบ→ปิดคำขอ (manual)Update stock
