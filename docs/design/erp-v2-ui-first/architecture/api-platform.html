Global search (ช่องค้นหาบน header) — US-PLT-04

ช่องเดียวบน header ("ค้นหาทั้งระบบ: PO / ลูกค้า / Lot / ใบแจ้งหนี้ ..."). พิมพ์แล้วค้น ข้ามหลาย entity พร้อมกัน คืนผลจัดกลุ่มตามชนิด + กดผล = deep link เข้า detail. กรองตามสิทธิ์ Read: entity ที่ user ไม่มี Read ของ module นั้น จะไม่โผล่ในผล.
GET/api/search?q=&limit=
ค้นแบบรวม (typeahead). ครอบ entity: customer (ชื่อ/บริษัท/เลขผู้เสียภาษี/contact), po (เลข PO), invoice (เลข INV), material (ชื่อ/รหัส), lot (เลข lot), batch (เลข batch), prd (เลข PRD), supplier (ชื่อ), bom (ชื่อสูตร), dn/shipment (เลข). คืน [{type, id, label, sub, deepLink}] จัดกลุ่มตาม type + จำกัดผลต่อกลุ่ม (เช่น 5) + "ดูทั้งหมดใน <module>". ผลถูกกรองด้วย Read permission ของ user (ต่อ module). ซ่อน soft-deleted (มี toggle "รวมที่ถูกลบ" เฉพาะ trace). ตามสิทธิ์ Read ราย module · <2s
index รองรับ: prefix/like index บนเลขเอกสารและชื่อ (เช่น po.id, invoice.id, customer.name, material.name, lot.id). ผลรวมจัดอันดับตามความตรง + ความใหม่.

Home — "งานที่รอฉัน" (task-inbox)

หน้าหลักไม่สร้างข้อมูลใหม่ — reuse จาก Notification + Dashboard: "งานที่รอฉัน" = noti ที่ยังไม่ ack ของ user + ทางลัดเข้าแผนกที่มีสิทธิ์. ให้ผู้ใช้มือใหม่เห็น "ต้องทำอะไรต่อ" ทันที.
GET/api/home
ประกอบจาก: ① งานที่รอฉัน = /api/notifications (unacknowledged, จัดกลุ่มตาม module) · ② ทางลัดแผนก = /api/dashboard/departments (แผนกที่มี Read) + ตัวเลขสรุปหลักต่อแผนก · ③ ชื่อ user + role. empty state: ไม่มีงานค้าง → "ไม่มีงานที่รอคุณ ✓". ทุก user (เห็นเฉพาะของตัวเองตามสิทธิ์)
ไม่มี endpoint ใหม่เชิงข้อมูล — Home เป็น aggregator ของ 2 endpoint ที่มีอยู่ (กัน logic ซ้ำ). หน้า home.

Session / บังคับ re-login 06:00

GET/api/auth/session
สถานะ session ปัจจุบัน: expiresAt (issued + 24h) + nextForcedResetAt (06:00 วันถัดไป) → UI ใช้ขึ้น คำเตือนก่อนหมดอายุ (P1-2ค) ให้ save งานก่อนถูกเตะออก. ผู้ล็อกอิน
POST/api/auth/login · POST/api/auth/google · POST/api/auth/logout
local (username/bcrypt) หรือ Google OIDC → ออก JWT (24h) + เขียน login_event (feed Admin dashboard "เข้าใช้" + trace). token valid เมื่อ iat ≥ session_epoch (เลื่อนทุก 06:00) — ADR-007.
