API Spec — BOM + Cost Snapshot

สร้างสูตรจากวัตถุดิบ (ค้นได้). ราคาทุน = สูงสุดของ supplier ที่ active คิด ณ ตอนสร้าง/แก้ + user แก้ทับได้ + snapshot ตอนบันทึก (ไม่คิดใหม่อัตโนมัติ) + badge เตือน "ราคาทุนอาจล้าสมัย" เมื่อราคา supplier ปัจจุบันต่างจาก snapshot. ราคาขาย บังคับกรอก.

อ่าน + badge สร้าง/แก้ + snapshot error

อ่าน + badge ล้าสมัย

GET/api/boms
รายการสูตร + pagination. แต่ละสูตรคืน cost_snapshot + cost_stale (bool = snapshot ต่างจาก max active supplier ปัจจุบัน) → UI ขึ้น badge. Read bom
GET/api/boms/{id}
รายละเอียด: ส่วนผสม (bom_line) + stock ปัจจุบันของแต่ละวัตถุดิบ (โชว์ประกอบ ไม่บังคับมี stock) + ราคาทุน snapshot vs ปัจจุบัน + ราคาขาย.

สร้าง / แก้ (snapshot ตอนบันทึก)

POST/api/boms
สร้างสูตร: เลือกวัตถุดิบ (ค้นได้) + จำนวน · คิด ราคาทุน = max(active supplier price) อัตโนมัติ (แก้ทับได้ ตั้ง cost_overridden) · snapshot ค่านี้ ตอน save · ราคาขายบังคับ. ถ้าวัตถุดิบไม่มี supplier active → บังคับกรอกราคาทุนเอง. Create bom
PATCH/api/boms/{id}
แก้สูตร → คิดราคาทุนใหม่ + snapshot ใหม่ (จนกว่า save ใหม่ จะยึด snapshot เดิม). audit ทุกช่อง.
DEL/api/boms/{id}
soft-delete: PO/PRD ที่ snapshot สูตรแล้ววิ่งต่อได้ · เปิด PO ใหม่ด้วยสูตรนี้ไม่ได้ (หายจาก dropdown). Delete bom · comment

error

codeเมื่อไร
BOM_SELL_PRICE_REQUIREDไม่กรอกราคาขาย (บังคับ)
BOM_COST_REQUIRED_NO_ACTIVE_SUPPLIERวัตถุดิบไม่มี supplier active + ไม่กรอกราคาทุนเอง
MATERIAL_REFERENCE_DELETEDใส่วัตถุดิบที่ถูกลบ
