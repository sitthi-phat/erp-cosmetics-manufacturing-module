API Spec — QC (ขาเข้า + แบตช์)

QC 2 แบบ: (1) ตรวจรับ Lot วัตถุดิบขาเข้า — ผ่าน→lot พร้อมใช้ / ไม่ผ่าน→lot ระงับ→ทำใบคืนของ. (2) ตรวจ Batch การผลิต — ผ่าน→PRD พร้อมส่งมอบ / ไม่ผ่าน→PRD Rework + feedback บังคับ. การตัดสิน QC ทำได้เฉพาะที่นี่ (หน้าผลิตไม่มีปุ่มตัดสิน). เห็น GMP chain: PO↔แบตช์↔Lot + ประวัติ run ก่อนหน้า.

คิว QC ตรวจ Lot ขาเข้า ตรวจ Batch error

คิว QC

GET/api/qc/queue
คิวรอตรวจ: ?target=lot|batch · lot รอตรวจรับ + batch รอ QC. เห็น PO/PRD/Lot ที่ผูก. Read qc
GET/api/qc/batches/{batchId}
รายละเอียดแบตช์ + วัตถุดิบ (lot ที่ใช้) + ประวัติ run ก่อนหน้า (rework) + feedback เดิม. GMP chain.

ตรวจรับ Lot วัตถุดิบขาเข้า

Endpointทำอะไร (cascade)สิทธิ์ / comment
POST /qc/lots/{lotId}/passLot รอตรวจรับ→พร้อมใช้ผลิต · (อาจปิด PR)Approve qc
POST /qc/lots/{lotId}/failLot→ระงับ · เสนอทำใบคืนของ (return) · noti StockApprove qc · feedback บังคับ

ตรวจ Batch การผลิต

Endpointทำอะไร (cascade)สิทธิ์ / comment
POST /qc/batches/{batchId}/passBatch→QC ผ่าน · PRD→พร้อมส่งมอบ · ถ้าทุก PRD ของ PO ผ่าน → PO→พร้อมจัดส่ง + noti Shipping (C5/C8)Approve qc
POST /qc/batches/{batchId}/failBatch→QC ไม่ผ่าน · PRD→Rework (กลับกำลังผลิต) · noti Production (C6)Approve qc · feedback บังคับ
ทุกการตัดสินเขียน qc_record + audit reason. Batch เป็นหลักฐาน GMP ห้ามลบ. Rework → ฝ่ายผลิตกด "ผลิตซ้ำ" gen Batch run ถัดไป (ดู api-production).
