API Spec — Dashboard + Drill-down

แดชบอร์ดราย 7 แผนก (ขาย/คลัง/ผลิต/QC/จัดส่ง/การเงิน/Admin). ทุก KPI tile drill-down ได้ → list + pagination + กดเข้า module พร้อม context. auto-refresh 15s (ค้าง view เดิม ผ่าน query param). date filter (วันนี้/สัปดาห์/เดือนนี้=default/กำหนดเอง) มีผลทุก tile. ทุก tile มี 2 endpoint: summary (นับ) + list (drill).

รูปแบบ tile (ใช้ทุกแผนก)

แต่ละ tile มี 2 endpoint คู่กัน เพื่อให้กด tile แล้วเปิด list ที่ filter ตรง context เดิม + กดรายการ deep link เข้า module. date filter + auto-refresh เก็บใน query param ไม่ใช่ state ฝั่ง server (refresh แล้วอยู่ view เดิม).
GET/api/dashboard/{dept}/summary?dateRange=
คืนตัวเลขทุก tile ของแผนก + caption (ในช่วง vs ตอนนี้). Read {dept}
GET/api/dashboard/{dept}/tiles/{tileKey}?page&dateRange=
drill-down: list + pagination ของ tile นั้น · แต่ละแถวมี deepLink เข้า module พร้อม context.

แผนก (dept)tiles (drill-down ได้ทุกตัว)
saleลูกค้าประจำ · ห่างหาย · ต้องติดตาม · PO เดือนนี้ · ค้างชำระ → list ลูกค้า/PO + เข้า detail
stockใกล้หมด · ล็อตรอ QC · คำขอ PR ค้าง · สต็อกติดลบ → list + เข้า module
productionคิวรอรับงาน · คิวงานผลิต · เสี่ยงล่าช้า · พักงาน (Hold) · เสร็จวันนี้
qcLot รอตรวจรับ · Batch รอ QC · ไม่ผ่านวันนี้
shippingPO พร้อมจัดส่ง · รอบกำลังนำส่ง · DN เลื่อน/ปฏิเสธ
financeรอวางบิล · ค้างชำระ · เกินกำหนด+จำนวนวัน · รับชำระวันนี้
adminผู้ใช้ · role · audit ล่าสุด · การตั้งค่าระบบ
user หลาย role → สลับแผนกได้. tile ทั้งหมดคือ drill-down ของ list endpoint ในแต่ละ module (ใช้ endpoint เดียวกับหน้า list ปกติ + filter) — ไม่สร้างข้อมูลซ้ำ.
