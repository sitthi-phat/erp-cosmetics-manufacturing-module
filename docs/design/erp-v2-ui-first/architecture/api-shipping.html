API Spec — Shipment (รอบ) + Delivery Note

โครงสร้าง 2 ชั้น: Shipment (รอบจัดส่ง) 1 รอบรวมหลาย DN, และ DN 1 ใบ = 1 PO เสมอ (ลูกค้าเซ็นรายใบ, print ทีละใบ). สร้างรอบจาก PO สถานะ "พร้อมจัดส่ง" (ค้นด้วย PO ID หรือข้อมูลลูกค้า). Reject→PO กลับคิว + raise Sale. Postpone→PO พร้อมจัดส่ง + flag กันจัดส่ง+วันที่ ค้างคิว. รอบจบเมื่อ DN ทุกใบถึงสถานะสุดท้าย.

สร้างรอบ (Shipment)

POST/api/shipping/rounds
สร้างรอบจาก PO ที่เลือก (พร้อมจัดส่งเท่านั้น) → gen 1 DN ต่อ 1 PO รวมเป็นรอบ · gen SHP + DN (gapless) · ข้อมูลรอบ: คนขับ/เบอร์/route/ประเภทรถ · สถานะรอบ = "รับเข้ารอบ". Create shipping
POST/api/shipping/rounds/{id}/dispatch
รอบ→กำลังนำส่ง · DN ทุกใบ→กำลังนำส่ง · PO→กำลังจัดส่ง (C9)
GET/api/shipping/rounds/{id}
รายละเอียดรอบ + DN ทุกใบ + สถานะ reconcile. รอบ→จบรอบ auto เมื่อ DN ทุกใบ terminal (C10c).
