API Spec — Invoice / ใบกำกับภาษีไทย

ออกใบแจ้งหนี้ได้ตั้งแต่ PO=Confirmed (แต่โชว์ stage จริงเสมอ). VAT ยึดเรตตาม effective date. เตือนค้างจ่าย (overdue) เมื่อส่งของแล้ว+เลยเครดิต+ยังไม่จ่าย. พิมพ์ใบกำกับภาษีไทยครบฟิลด์ (ผู้ออกจาก settings, ตัวหนังสือไทย, ลายเซ็น 2 ช่อง). มี versioning.

อ่าน / ค้นหา

GET/api/invoices
รายการ + pagination · filter สถานะ (รอชำระ/ชำระ/เกินกำหนด) · โชว์ stage ของ PO · จำนวนวันค้าง. Read invoice
GET/api/invoices/{id}
รายละเอียด + versioning + trace. หน้า invoice-detail
GET/api/invoices/{id}/print
ข้อมูลครบสำหรับพิมพ์ A4: company_profile (ผู้ออก+เลขผู้เสียภาษี), รายการ, subtotal/discount/VAT ตาม effective/grand total, ตัวหนังสือไทย. หน้า invoice-print

ออก / รับชำระ

Endpointทำอะไรสิทธิ์
POST /invoices (body: po_id)ออกใบแจ้งหนี้ (ตั้งแต่ PO=Confirmed) · gen INV-{YYYY}-{6} (gapless) · snapshot VAT rate ตาม effective ณ วันออก · billing→วางบิลCreate invoice
POST /invoices/{id}/reviseแก้ → สร้าง version ใหม่ (เก็บรุ่นเก่า + audit)Update invoice
POST /invoices/{id}/payรับชำระครบ → billing=ชำระแล้วApprove invoice
POST /invoices/{id}/voidยกเลิก (ไม่ลบ คงเลข) + comment บังคับ → ออกใบใหม่แทนAdmin invoice
overdue เป็น auto จาก scheduler (ส่งของแล้ว + เลย credit_days ของลูกค้า + ยังไม่จ่าย) → ยิง noti Finance + Sale (cascade #14).

VAT + effective date

เรต VAT เก็บเป็นประวัติใน vat_config (rate + effective_date). ตอนออกใบแจ้งหนี้ ระบบเลือกเรตที่ "มีผล ณ วันออก" แล้ว snapshot ลงใบ — เปลี่ยนเรตภายหลังไม่กระทบใบเก่า. Admin ตั้งเรต+วันมีผลที่ settings.

error

codeเมื่อไร
INVOICE_PO_NOT_CONFIRMEDออกใบก่อน PO ยืนยัน
VAT_CONFIG_MISSINGไม่มีเรต VAT ที่มีผล ณ วันออก
INVOICE_VOID_REASON_REQUIREDยกเลิกโดยไม่ใส่เหตุผล
