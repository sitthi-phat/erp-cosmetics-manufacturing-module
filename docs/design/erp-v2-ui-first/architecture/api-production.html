API Spec — Production (PRD + Batch)

หัวใจของการผลิต: PO ยืนยัน → คิว "รอรับงาน" → ฝ่ายผลิตกด "รับงาน" (เกิดเลข PRD) → "เริ่มผลิต" (เกิด Batch + ตัดสต็อก ติดลบได้) → ส่ง QC → พร้อมส่งมอบ. 1 PO : N PRD (N=line) : M Batch. QC ตัดสินที่ module QC เท่านั้น (หน้านี้ไม่มีปุ่มตัดสิน QC).

สต็อกติดลบ (ตอน start)

ตอน start ระบบ convert reservation → consume จริง (reserved ลด + ตัด on_hand FIFO). ถ้าวัตถุดิบไม่พอ ระบบ ไม่บล็อก — เขียน stock_movement qty ลบ (lot_id อาจ null ถ้ายังไม่มี lot) ตั้ง caused_negative=true + trace. หน้า stock โชว์ตัวเลขแดง + badge "ติดลบ (รอรับเข้า)". พอทำ Goods Receipt → retro-alloc ผูก lot FIFO + บวกกลับ. รายละเอียด ADR-001.
