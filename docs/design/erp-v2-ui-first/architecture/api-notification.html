API Spec — Notification + Deep Link

ทุกการส่งงานข้าม module ยิงแจ้งเตือน. ผู้รับ = ทุกคนที่มีสิทธิ์ Read ของ module ปลายทาง. bell มุมขวาบน → badge รวม → expand เป็นรายการ → กดรายการ = deep link ไปหน้าทำงานต่อ + acknowledge (badge ลดราย user). ใช้ transactional outbox กันหลุด (ADR-005).

bell / รายการ

GET/api/notifications
รายการ noti ที่ยังไม่ ack ของ user (derive จากสิทธิ์ Read + ลบด้วย notification_ack) · แต่ละรายการมี deepLink (route+params) · badge count. polling ~15s (ADR-005). ตามสิทธิ์ Read ราย module
GET/api/notifications/badge
เลขรวม badge + แยกราย module (สำหรับ badge เมนูซ้าย).

acknowledge

POST/api/notifications/{id}/ack
กดอ่าน 1 รายการ → insert notification_ack (idempotent) → badge ลด (5→4). ปกติเรียกพร้อม navigate ตาม deepLink. ผู้มีสิทธิ์ Read ของ module นั้น
