Deletion Policy — 7 กติกากลาง

อ้าง deletion-policy.md · ทั้งระบบไม่มี hard delete

  1. Soft delete เท่านั้น — ตั้ง flag deleted + เก็บ record ไว้
  2. ยัง search/เปิดดูย้อนหลังได้ (read-only, badge "ถูกลบ")
  3. ห้ามเป็น reference ของของใหม่ — หายจาก dropdown/ตัวเลือกงานใหม่
  4. ของเดิมที่ผูกอยู่แล้ววิ่งต่อได้ — ไม่ break integrity ย้อนหลัง
  5. Audit บังคับ — ใครลบ/เมื่อไหร่/เหตุผล (comment บังคับ)
  6. สิทธิ์: ลบ = bit D · กู้คืน = bit Admin · บาง entity ต้อง Approve/Manager
  7. เอกสารการค้า (PO/Invoice/DN/GR/PR/Shipment) ไม่มีปุ่มลบ — ใช้ "ยกเลิก/void" (เลข gapless ห้ามหาย)
