Module: การผลิต — PRD / Batch Tier 1

ใบสั่งผลิต (PRD) 1 ใบ/1 line · รุ่นผลิต (Batch) B-{PO}-{line}-{run} · ฝ่ายผลิตกด "รับงาน" เอง จึงเกิด PRD · เลข PRD PRD-{YYYYMM}-{NNNNNN}

เส้นทางหลัก: PO ยืนยัน → คิว "รอรับงาน" (ยังไม่มี PRD) →[Production กด "รับงาน"]→ gen PRD (รับงาน) →[กด "เริ่มผลิต"]→ gen Batch run1 + ตัด stock (ติดลบได้) →[ส่งตรวจ QC]→ รอ QC →[QC ตัดสินที่หน้า QC]→ ผ่าน = PRD พร้อมส่งมอบ / ไม่ผ่าน = Rework (gen Batch run ถัดไป) · ทุก PRD ของ PO พร้อม → PO พร้อมจัดส่ง
สำคัญ: หน้าผลิต ไม่มีปุ่มตัดสิน QC (ผ่าน/ไม่ผ่านตัดสินที่หน้า QC เท่านั้น) · "ส่งตรวจ QC" เปลี่ยนสถานะ Batch เดิม ไม่ได้ gen Batch ใหม่ · หน้าผลิตจบที่ "พร้อมส่งมอบ" ("ส่งถึงแล้ว" เป็นของฝ่ายจัดส่ง)

1. ภาพรวม + Use cases

ฝ่ายผลิตรับงานจากคิว, เริ่มผลิต (สร้าง Batch + ตัด stock), ส่งตรวจ QC, และจัดการ Hold/Rework · วัตถุดิบขาดไม่บล็อก — รับงาน/เริ่มผลิตได้แม้ stock ไม่พอ (ตัด stock ติดลบ, badge เตือน)

โผล่ใน: UC3 (QC ผ่าน PO-176), UC4 (rework PO-181), UC5 (rework ครบวง PO-170), UC2 (Hold PO-188)

3. User Stories + Acceptance Criteria

US-PRD-01 — รับงานจากคิว (gen PRD) Must

As a ฝ่ายผลิต, I want กด "รับงาน" ที่รายการคิว "รอรับงาน", so that ยืนยันรับงานเข้าสายผลิตและระบบออกเลขใบสั่งผลิต (PRD)

Happy Given PO-202607-000181 ยืนยันแล้ว มี 2 line อยู่ในคิว "รอรับงาน" (ยังไม่มีเลข PRD) When ฝ่ายผลิตกด "รับงาน" ที่ line 1 Then ระบบ gen PRD-202607-000091 สถานะ "รับงาน" (1 line = 1 PRD), แสดงในหน้า production + po-detail (คอลัมน์ PRD), มี trace "รับงานโดย <user>/เวลา" — ดู Cascade C2
Edge — วัตถุดิบไม่พอ (ไม่บล็อก) Given วัตถุดิบของ line นี้มียอดคงเหลือน้อยกว่าที่ BOM ต้องใช้ When กด "รับงาน" Then ระบบสร้าง PRD ได้ตามปกติ + แสดง badge/เตือน "วัตถุดิบอาจไม่พร้อม" (ไม่บล็อก) — ผลิตต่อได้
Error — ไม่มีสิทธิ์ / PO ถูกยกเลิก Given ผู้ใช้ไม่มีสิทธิ์ Update/Create ของ module ผลิต หรือ PO ถูกยกเลิกไปแล้ว When พยายามรับงาน Then ปุ่มรับงานถูก disable / คืน error "รายการนี้ถูกยกเลิก" — ไม่ออกเลข PRD

US-PRD-02 — เริ่มผลิต (gen Batch + ตัด stock ติดลบได้) Must

As a ฝ่ายผลิต, I want กด "เริ่มผลิต" ของ PRD, so that ระบบสร้าง Batch run และตัดวัตถุดิบตาม BOM (แบบ FIFO)

Happy Given PRD-202607-000091 สถานะ "รับงาน" และวัตถุดิบพอ When กด "เริ่มผลิต" Then gen B-PO-202607-000181-1-1 (กำลังผลิต); ตัด stock วัตถุดิบตาม BOM โดยเลือก Lot FIFO; PRD = "กำลังผลิต"; PO = "กำลังผลิต" — ดู Cascade C3
Edge — stock ติดลบ (negative stock) Given ยอดคงเหลือวัตถุดิบไม่พอสำหรับ Batch นี้ When กด "เริ่มผลิต" Then ระบบยอมให้ตัด stock จนติดลบ (ไม่บล็อก) + เตือน "จะตัด stock ติดลบ X หน่วย"; บันทึก trace (Batch/วัตถุดิบ/จำนวนติดลบ/เวลา/ใคร); หน้า stock แสดงตัวเลขแดง + badge "ติดลบ (รอรับเข้า)" — ดู Cascade C16 · Stock
Error — PRD ไม่อยู่สถานะรับงาน Given PRD อยู่สถานะ Hold หรือ พร้อมส่งมอบแล้ว When พยายามกด "เริ่มผลิต" Then ปุ่มไม่ทำงาน / error "ไม่สามารถเริ่มผลิตในสถานะปัจจุบัน"; ไม่ gen Batch ซ้ำ

US-PRD-03 — ส่งตรวจ QC Must

As a ฝ่ายผลิต, I want ส่ง Batch ที่ผลิตเสร็จไปตรวจคุณภาพ, so that QC ตัดสินผ่าน/ไม่ผ่าน

Happy Given Batch B-…-181-1-1 สถานะ "กำลังผลิต" When กด "ส่งตรวจ QC" Then Batch เดิมเปลี่ยนเป็น "รอ QC" (ไม่ gen Batch ใหม่); PRD = "รอ QC"; โผล่คิวหน้า QC; ยิง Notification → QC — ดู Cascade C4
Edge — หลาย line/หลาย Batch Given PO มีหลาย line กำลังผลิตพร้อมกัน When ส่งตรวจเฉพาะ Batch ของ line 2 Then เฉพาะ Batch/PRD ของ line 2 เปลี่ยนเป็น "รอ QC"; line อื่นคงสถานะเดิม (แยกอิสระราย line)
Error — ส่งตรวจซ้ำ Given Batch อยู่สถานะ "รอ QC" อยู่แล้ว When กด "ส่งตรวจ QC" อีกครั้ง Then ระบบไม่สร้างรายการซ้ำ + แจ้ง "อยู่ระหว่างตรวจ QC แล้ว"

US-PRD-04 — Rework เมื่อ QC ไม่ผ่าน Must

As a ฝ่ายผลิต, I want เห็น feedback และผลิตซ้ำเฉพาะ line ที่ QC ตีกลับ, so that แก้เฉพาะตัวที่เสียโดยไม่ผลิตซ้ำ line ที่ผ่าน/ยังไม่เสร็จ

Happy Given QC ตัดสิน B-…-181-2-1 (โฟม) "ไม่ผ่าน" + feedback "เนื้อโฟมเป็นก้อน" When ฝ่ายผลิตเปิดหน้าผลิตแล้วกด "ผลิตซ้ำ" Then line 2 badge "Rework" + แสดง feedback จาก QC; gen B-…-181-2-2 (run ถัดไป, กำลังผลิต); line 1 เดินต่อปกติ — ดู Cascade C7
Edge — reconcile PO ไม่พร้อม Given line 1 รอ QC, line 2 กำลัง rework When ดู po-detail Then แสดง "ผ่าน 0/2" — PO ยังไม่ พร้อมจัดส่งจนกว่าทุก line ผ่าน
Error — ผลิตซ้ำโดยยังไม่มี feedback Given Batch ยังไม่ถูก QC ตัดสินไม่ผ่าน When พยายามกด "ผลิตซ้ำ" Then ปุ่มไม่แสดง (มีเฉพาะ Batch ที่ QC ตีกลับพร้อม feedback บังคับ)

US-PRD-05 — พักงาน (Hold) + raise Must

As a ฝ่ายผลิต, I want พักงานพร้อมระบุเหตุและ raise ไป Sale หรือ Stock, so that ปัญหาที่ติดคน/วัตถุดิบถูกส่งให้ผู้รับผิดชอบแก้

Happy Given PRD ของ PO-188 กำลังผลิตแต่ลูกค้าขอแก้สูตร When กด "พักงาน (Hold)" + เลือก raise "แจ้ง Sale (ติดที่ลูกค้า)" + เหตุผล (บังคับ) Then PRD = "Hold"; ยิง Notification → Sale (deep link po-detail); Sale แก้ไข PO ได้ (ดู US-PO-05) — ดู Cascade C8
Edge — Hold ติดวัตถุดิบ Given ติดปัญหาวัตถุดิบ When เลือก raise "แจ้ง Stock" Then Notification → Stock; หลังแก้เสร็จ ฝ่ายผลิตปลด Hold กลับ "กำลังผลิต" ต่อได้
Error — Hold โดยไม่ใส่เหตุผล Given ช่องเหตุผลว่าง When กดยืนยัน Hold Then ไม่ Hold + error "ต้องระบุเหตุผลการพักงาน" (comment บังคับ)

US-PRD-06 — เสี่ยงล่าช้า (Potential Delay) + เรียง/ค้นวันส่ง Should

As a ฝ่ายผลิต/หัวหน้า, I want เห็น overlay "เสี่ยงล่าช้า" และเรียง/ค้นด้วยวันจัดส่ง/PO/ลูกค้า, so that จัดลำดับงานผลิตได้

Happy Given งานที่เวลาคงเหลือน้อยกว่าเกณฑ์ (2 วันผลิต + 1 วันส่ง) When เปิดหน้าผลิต Then รายการนั้นมี badge overlay "เสี่ยงล่าช้า" (ไม่ใช่สถานะแยก) + ยิง Notification → Sale/Stock (ref)
Edge — เรียง/ค้น Given มีหลายงานในคิว When เรียงตาม "วันจัดส่ง" หรือค้นด้วยเลข PO/ชื่อลูกค้า Then รายการถูกจัดเรียง/กรองตามเงื่อนไข
Error — เกณฑ์คำนวณไม่ได้ Given งานที่ยังไม่มีวันจัดส่งที่ต้องการ When ประเมิน Potential Delay Then ไม่ติด badge เสี่ยงล่าช้า (ข้อมูลไม่ครบ) + ไม่ error
