3. User Stories + Acceptance Criteria

US-PLT-01 — เข้าสู่ระบบ + identity + session warning ก่อน 06:00 Must

As a พนักงาน, I want ล็อกอิน เห็นชื่อระบบ/โลโก้ทุกหน้า และได้รับเตือนก่อน session หมด/reset, so that ใช้งานได้มั่นใจและไม่เสียงานค้างตอน session ตัด

Happy Given ผู้ใช้มี account When ล็อกอิน (local หรือ Google) Then เข้าระบบ; ทุกหน้าแสดง "ESSENCE Hub System" + logo/icon และ browser title; เห็นเมนู/ค้นหา/dashboard ตามสิทธิ์ Read
Edge — เตือนก่อน session reset 06:00 Given ใกล้เวลา reset session ประจำวัน 06:00 (หรือ session ใกล้ครบ 24 ชม.) When เหลือเวลาก่อนตัด (เช่น 5 นาที) Then แสดงแถบเตือน "ระบบจะออกจากระบบเวลา 06:00 กรุณาบันทึกงาน" ล่วงหน้า; เมื่อถึงเวลา → ให้ล็อกอินใหม่ (ไม่ทำงานค้างเงียบๆ)
Error — รหัสผิด Given กรอกรหัสผิด When ล็อกอิน Then error "ชื่อผู้ใช้หรือรหัสผ่านไม่ถูกต้อง" — ไม่เข้าระบบ (ไม่บอกว่าผิดที่ช่องไหนเพื่อความปลอดภัย)

US-PLT-02 — Notification: routing ตาม Read-bit + deep link + ack ราย user Must

As a ผู้ใช้ที่มีสิทธิ์ Read ของ module, I want รับแจ้งเตือนงานส่งข้ามแผนกและกดไปทำต่อ, so that ไม่พลาดงานและไปทำงานต่อได้ทันที

Happy Given มี 5 การแจ้งเตือน (เช่น PO-181 Confirmed → Production) When กด bell → กดรายการ "PO-181 เข้าคิวผลิต" Then deep link ไป production พร้อม context + acknowledge รายการนั้น → badge เหลือ 4 (ราย user)
Edge — routing ตาม Read permission Given เหตุการณ์เข้า module Production When ยิง notification Then เฉพาะผู้มีสิทธิ์ Read ของ Production ได้รับ (ผู้อื่นไม่เห็น) — ดู Notification matrix
Error — ack ของคนอื่น Given ผู้ใช้ A กด ack รายการ When ผู้ใช้ B เปิด bell Then badge/สถานะ ack เป็น ราย user (A ack ไม่ทำให้ของ B หาย)

US-PLT-03 — Notification panel: grouping / mark all read / empty / badge cap / ดูทั้งหมด Must

As a ผู้ใช้, I want แผงแจ้งเตือนอ่านง่าย จัดกลุ่ม เคลียร์ทั้งหมดได้ และไปหน้ารวมได้, so that จัดการแจ้งเตือนจำนวนมากได้โดยไม่รก

Happy — ลำดับ + grouping + ดูทั้งหมด Given เปิด bell panel มีแจ้งเตือน 8 รายการ When ดู panel Then รายการเรียงใหม่→เก่า, จัดกลุ่ม "ยังไม่อ่าน" ก่อน; แสดงสูงสุด N รายการ + ลิงก์ "ดูทั้งหมด" ไปหน้ารวมแจ้งเตือน (list + pagination)
Edge — mark all read + badge cap "9+" Given ยังไม่อ่าน 12 รายการ (badge แสดง "9+") When กด "ทำเป็นอ่านทั้งหมด (mark all read)" Then ทุกรายการของ user นี้เป็นอ่านแล้ว, badge → 0 (ไม่กระทบ user อื่น); ค่า badge > 9 แสดงเป็น "9+" เสมอ
Edge/Empty — ไม่มีแจ้งเตือน Given ไม่มีแจ้งเตือนค้าง When เปิด panel Then แสดง empty state "ไม่มีการแจ้งเตือน" (ไม่ error, ไม่ panel ว่างเปล่างงๆ); badge ไม่แสดงตัวเลข

US-PLT-04 — Global header search (ค้นข้าม module ตามสิทธิ์) Should

As a ผู้ใช้, I want ค้นหาเอกสาร/ลูกค้า/วัตถุดิบจากช่องค้นบน header แล้วกดไปหน้ารายละเอียด, so that หาสิ่งที่ต้องการได้เร็วจากทุกหน้า

Happy — ค้นได้อะไร + คลิกไปไหน Given ผู้ใช้พิมพ์ "181" หรือ "กลอรี่" ในช่องค้น header When ระบบค้น Then แสดงผลจัดกลุ่มตามชนิด (PO / ลูกค้า / วัตถุดิบ-Lot / PR-GR / DN-Invoice) เช่น "PO-202607-000181", "ลูกค้า กลอรี่"; แต่ละผลคลิก → deep link ไปหน้ารายละเอียดของ entity นั้น
Edge — จำกัดตามสิทธิ์ Read + ไม่พบผล Given ผู้ใช้ไม่มีสิทธิ์ Read module ของบางผลลัพธ์ หรือคำค้นไม่ตรงอะไร When ค้น Then ผลที่อยู่นอกสิทธิ์ Read ไม่ปรากฏ; ถ้าไม่พบ → "ไม่พบผลการค้นหา" (ไม่ error)
Error — คำค้นสั้นเกินไป Given พิมพ์ตัวอักษรน้อยกว่า 2 ตัว When ค้น Then ไม่ยิงค้น + ข้อความช่วยเหลือ "พิมพ์อย่างน้อย 2 ตัวอักษร" (กัน query หนักโดยเปล่าประโยชน์)

US-PLT-05 — Responsive + RBAC guard ทุกหน้า Must

As a ผู้ใช้, I want ใช้งานได้ทุกอุปกรณ์ และเห็นเฉพาะสิ่งที่มีสิทธิ์, so that ทำงานได้จริงและปลอดภัย

Happy Given เปิดบน mobile/tablet/desktop When ใช้งาน Then layout responsive (hamburger/mobile tab-bar) ทุกหน้า; ไม่มีหน้าโชว์ enum/รหัสดิบ (สถานะเป็นป้ายไทย)
Edge — เมนูตามสิทธิ์ Given ผู้ใช้ไม่มีสิทธิ์ Read module X When ดูเมนูซ้าย Then ไม่เห็นเมนู X + เข้า URL ตรงถูกปฏิเสธ (guard 403)
Error — action เกินสิทธิ์ Given มีสิทธิ์ Read แต่ไม่มี Create When พยายามสร้างข้อมูล Then ปุ่มสร้างถูกซ่อน/disable; ถ้าเรียก API ตรง → 403
