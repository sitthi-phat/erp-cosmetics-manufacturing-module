3. User Stories + Acceptance Criteria

US-PLT-01 — เข้าสู่ระบบ + identity Must

As a พนักงาน, I want ล็อกอินและเห็นชื่อระบบ/โลโก้ทุกหน้า, so that ใช้งานได้อย่างมั่นใจ

Happy Given ผู้ใช้มี account When ล็อกอิน (local หรือ Google) Then เข้าระบบ; ทุกหน้าแสดง "ESSENCE Hub System" + logo/icon (browser title ด้วย); เห็นเมนูตามสิทธิ์
Edge — session หมดอายุ/reset 06:00 Given session เกิน 24 ชม. หรือถึง 06:00 (reset ทั้งระบบ) When ใช้งานต่อ Then ระบบให้ล็อกอินใหม่ (ไม่ทำงานค้าง)
Error — รหัสผิด Given กรอกรหัสผิด When ล็อกอิน Then error "ชื่อผู้ใช้หรือรหัสผ่านไม่ถูกต้อง" — ไม่เข้าระบบ

US-PLT-02 — Notification bell + deep link + acknowledge Must

As a ผู้ใช้ที่มีสิทธิ์ Read ของ module, I want รับแจ้งเตือนงานส่งข้ามแผนกและกดไปทำต่อ, so that ไม่พลาดงานและไปทำงานต่อได้ทันที

Happy Given มี 5 การแจ้งเตือน (เช่น PO Confirmed → Production) When กด bell → กดรายการ "PO-181 เข้าคิวผลิต" Then deep link ไปหน้า production พร้อม context + acknowledge รายการนั้น → badge เหลือ 4 (ราย user)
Edge — routing ตาม Read permission Given เหตุการณ์เข้า module Production When ยิง notification Then เฉพาะผู้ที่มีสิทธิ์ Read ของ Production ได้รับ (ผู้อื่นไม่เห็น) — ดู Notification matrix
Error — ack ของคนอื่น Given ผู้ใช้ A กด ack รายการ When ผู้ใช้ B เปิด bell Then badge/สถานะ ack เป็น ราย user (A ack ไม่ทำให้ของ B หาย)

US-PLT-03 — Responsive + RBAC guard ทุกหน้า Must

As a ผู้ใช้, I want ใช้งานได้ทุกอุปกรณ์ และเห็นเฉพาะสิ่งที่มีสิทธิ์, so that ทำงานได้จริงและปลอดภัย

Happy Given เปิดบน mobile/tablet/desktop When ใช้งาน Then layout responsive (hamburger/mobile tab-bar) ทุกหน้า; ไม่มีหน้าโชว์ enum/รหัสดิบ
Edge — เมนูตามสิทธิ์ Given ผู้ใช้ไม่มีสิทธิ์ Read module X When ดูเมนูซ้าย Then ไม่เห็นเมนู X + เข้า URL ตรงถูกปฏิเสธ (guard)
Error — action เกินสิทธิ์ Given มีสิทธิ์ Read แต่ไม่มี Create When พยายามสร้างข้อมูล Then ปุ่มสร้างถูกซ่อน/disable; ถ้าเรียก API ตรง → 403
