3. User Stories + Acceptance Criteria

US-CUS-01 — สร้างลูกค้า + contact ไม่จำกัด + ฟิลด์ภาษี Must

As a Sale, I want สร้างลูกค้าพร้อมเลขผู้เสียภาษี 13 หลัก ที่อยู่จดทะเบียน และเพิ่ม contact ได้หลายคน, so that ออกใบกำกับภาษีและติดต่อได้ครบ

Happy Given Sale อยู่หน้า customer-create When กรอกชื่อ+เลขผู้เสียภาษี 13 หลัก+ที่อยู่ แล้วบันทึก Then สร้าง CUS-xxxxxx สถานะ "ผู้สนใจ (Lead)"; เพิ่ม contact ได้ไม่จำกัด
Edge — contact หลายคน Given ลูกค้ามี contact 3 คน When เพิ่ม contact ที่ 4 Then บันทึกได้ (ไม่จำกัดจำนวน) + ระบุผู้ติดต่อหลักได้
Error — เลขภาษีไม่ครบ 13 หลัก Given กรอกเลขผู้เสียภาษี ≠ 13 หลัก When บันทึก Then error "เลขผู้เสียภาษีต้อง 13 หลัก" — ไม่บันทึก

US-CUS-02 — วงจรสถานะ auto (Active↔Inactive) Should

As a ระบบ/Sale, I want สถานะเปลี่ยนอัตโนมัติตามรอบ order, so that รู้ว่าลูกค้าประจำหรือห่างหายโดยไม่ต้องอัปเดตมือ

Happy Given ลูกค้า Lead ยืนยัน PO ใบแรก When PO Confirmed Then ลูกค้า → "ลูกค้าประจำ (Active)" auto (C1)
Edge — ไม่มี order เกินรอบ Given config รอบ = 3 เดือน และลูกค้าไม่มี order เกิน 3 เดือน When scheduler ประเมิน Then ลูกค้า Active → "ห่างหาย (Inactive)" auto + แจ้ง Sale; มี order ใหม่ → กลับ Active
Error — config รอบผิด Given ตั้งค่ารอบเป็นค่าที่ไม่อยู่ใน {1,3,6,8} เดือน When บันทึก config ต่อลูกค้า Then error "ค่ารอบไม่ถูกต้อง" — ไม่บันทึก

US-CUS-03 — ตั้ง "ต้องติดตาม" (Follow-up) + comment บังคับ Must

As a Sale/Sale Manager, I want ตั้งลูกค้าเป็น "ต้องติดตาม" พร้อมเหตุผล, so that ทีมขายรู้ว่าต้องติดตามเรื่องอะไร (เช่น งานผลิตติด Hold)

Happy Given เดอร์มา แคร์ มีงาน PO-188 ติด Hold (raise มาที่ Sale) When Sale เปลี่ยนสถานะเป็น "ต้องติดตาม" + comment "โทรยืนยันการแก้สูตร PO-188" Then ลูกค้า → Follow-up (mutually exclusive กับ Active/Inactive); โผล่ tile "ต้องติดตาม" ใน Sale Dashboard; trace บันทึก (C8)
Edge — ปิดการติดตาม Given ลูกค้า Follow-up จัดการเรื่องจบ When Sale ปิดการติดตาม หรือมี order ใหม่ Then กลับเป็น Active (มี order) หรือ Inactive (ยังไม่มี order)
Error — ไม่ใส่ comment Given ช่อง comment ว่าง When กดตั้ง Follow-up Then ไม่เปลี่ยนสถานะ + error "ต้องระบุเหตุผลการติดตาม" (comment บังคับ)

US-CUS-04 — Disable/Blacklist + reassign Should

As a Sale Manager/Admin, I want ปิดใช้งาน/ขึ้นบัญชีดำลูกค้า และย้ายผู้ดูแล (reassign), so that ควบคุมความเสี่ยงและกระจายงานได้

Happy Given ลูกค้าเช็คเด้ง 2 ครั้ง When Sale Manager ตั้ง "บัญชีดำ" + comment (บังคับ) Then ลูกค้า → Blacklist; หายจาก dropdown ตอนเปิด PO ใหม่ + เตือนเมื่อพยายามเปิด PO
Edge — reassign Given Sale Manager ต้องการย้ายลูกค้าจากสมชายไปสมหญิง When กด reassign Then customer.sale เปลี่ยน; แสดง 2 dashboard (เดิม/ใหม่) + trace (ref)
Error — ปลด Blacklist โดยไม่ใช่ Admin Given ผู้ใช้ไม่มีสิทธิ์ Admin When พยายามปลด Blacklist Then ไม่มีตัวเลือก/403 (ปลด Blacklist = Admin เท่านั้น) — ดู RUCDAA
