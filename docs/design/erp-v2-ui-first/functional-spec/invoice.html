3. User Stories + Acceptance Criteria

US-INV-01 — ออกใบแจ้งหนี้ตั้งแต่ Confirmed Must

As a Finance, I want ออกใบแจ้งหนี้ตั้งแต่ PO ยืนยันแล้ว แต่เห็น stage จริงของ PO, so that วางบิลได้เร็วโดยไม่หลุดสถานะงานจริง

Happy Given PO-202607-000176 สถานะ "ส่งถึงแล้ว" When Finance กด "ออกใบแจ้งหนี้" Then gen INV-2026-000135 สถานะ "รอชำระ"; ราง billing ของ PO = "วางบิลแล้ว"; แสดง PO fulfilment stage = "ส่งถึงแล้ว"; เครดิต 30 วัน (จากระดับลูกค้า) — ดู Cascade C13
Edge — ออกตั้งแต่ Confirmed Given PO เพิ่งยืนยัน (ยังไม่ผลิต/ส่ง) When ออกใบแจ้งหนี้ Then อนุญาต + ใบแสดงป้ายเตือน PO stage = "ยืนยันแล้ว (ยังไม่ส่ง)" ชัดเจน
Error — PO ยังเป็นร่าง/ยกเลิก Given PO สถานะ "ร่าง" หรือ "ยกเลิก" When พยายามออกใบแจ้งหนี้ Then ระบบบล็อก + error "ออกใบแจ้งหนี้ได้ตั้งแต่ PO ยืนยันแล้วเท่านั้น"

US-INV-02 — ใบกำกับภาษีไทย + VAT effective date Must

As a Finance/ลูกค้า, I want ใบกำกับภาษีไทยครบฟิลด์และคิด VAT ตามอัตราที่มีผล ณ วันออกใบ, so that เอกสารถูกต้องตามสรรพากร

Happy Given ตั้งค่า VAT 7% effective 01/01/2569 ใน settings และออกใบวันที่ 08/07/2569 When เปิด invoice-print Then ใบแสดง logo, ผู้ออก+เลขผู้เสียภาษี 13 หลัก, ลูกค้า, เลขที่/วันที่/เครดิต, ตารางรายการ, subtotal, discount, VAT 7% (มีผล 01/01/2569), grand total, ตัวหนังสือไทย, ลายเซ็น 2 ช่อง
Edge — มีหลายอัตรา VAT Given มีการตั้งค่า VAT หลายช่วง effective date When ออกใบวันที่ที่อยู่ในช่วงใด Then ระบบเลือกอัตราที่ effective ครอบคลุม วันที่ออกใบ (invoice date) เท่านั้น
Error — ข้อมูลบริษัทไม่ครบ Given Admin ยังไม่กรอกข้อมูลบริษัท/เลขผู้เสียภาษีใน settings When พยายามพิมพ์ใบกำกับภาษี Then เตือน "ข้อมูลผู้ออกไม่ครบ กรุณากรอกในตั้งค่า" — ไม่พิมพ์ใบที่ฟิลด์บังคับว่าง

US-INV-03 — Overdue / ติดตามหนี้ Should

As a Finance/Sale, I want เห็นใบที่เกินกำหนดชำระพร้อมจำนวนวันค้าง, so that ติดตามเก็บเงินได้

Happy Given ใบที่ PO ส่งถึงแล้ว + เลยเครดิต (ระดับลูกค้า) + ยังไม่จ่าย When scheduler ประเมินรายวัน Then billing = "เกินกำหนด (Overdue)" + แสดงจำนวนวันค้าง; ยิง Notification → Finance + Sale — ดู Cascade C14
Edge — ยังไม่ส่งของ Given ออกใบแล้วแต่ PO ยังไม่ "ส่งถึงแล้ว" When เลยวันเครดิต Then ยังไม่ ถือ overdue (นับ overdue เมื่อส่งของแล้วเท่านั้น)
Error — รับชำระเกิน/ผิด Given บันทึกรับชำระมากกว่ายอดใบ When ยืนยัน Then error "ยอดรับชำระเกินยอดค้าง" — ไม่บันทึก

US-INV-04 — Versioning / ยกเลิก (void) Should

As a Finance, I want แก้ไข/ยกเลิกใบแบบมีเวอร์ชัน (ไม่ลบ), so that เลขเอกสารครบ gapless และตรวจย้อนได้

Happy Given ใบ INV มีการแก้ไข When บันทึกการแก้ Then เก็บเป็นเวอร์ชันใหม่ + เห็นประวัติเวอร์ชัน; เลขใบเดิมคงอยู่
Edge — void Given ต้องยกเลิกใบ When กด "ยกเลิก (void)" + เหตุผล Then ใบ = void (ไม่ลบออกจากระบบ), เลขคงอยู่ (gapless), ออกใบใหม่แทนได้ — ดู Deletion §2.8
Error — ไม่มีปุ่มลบถาวร Given ผู้ใช้พยายามลบใบถาวร When เปิดใบ Then ไม่มีปุ่ม "ลบ" — มีเฉพาะ "ยกเลิก/void" (เอกสารการค้าห้าม hard delete)

4. Data rules / สถานะ Entity

รายการกติกา
เลข InvoiceINV-{YYYY}-{NNNNNN} gapless ต่อปี
Billing statusรอชำระ (Invoiced) → ชำระแล้ว (Paid) / เกินกำหนด (Overdue)
VATอัตราตาม effective date ยึด invoice date · effective date ตั้งใน settings
เครดิต (overdue)จำนวนวันเครดิตจากระดับลูกค้า (default) แก้ต่อใบได้ · นับ overdue เมื่อส่งของแล้ว
ใบกำกับภาษีฟิลด์ครบ (§5 brief) + ตัวหนังสือไทย + ลายเซ็น 2 ช่อง · ข้อมูลผู้ออกจาก settings
ลบไม่มี hard delete — void เท่านั้น (gapless)
