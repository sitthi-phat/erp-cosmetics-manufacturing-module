Module: จัดส่ง — Shipment (รอบ) + DN Should

2 ชั้น: Shipment (รอบจัดส่ง) รวมหลาย DN · DN = 1 ใบต่อ 1 PO เสมอ (ลูกค้าเซ็นรายใบ) · เลข SHP-{YYYYMMDD}-{NNNN} / DN-{YYYYMMDD}-{NNNNN}

1. ภาพรวม + Use cases

หน้าจัดส่ง = สร้างรอบ (เลือก PO "พร้อมจัดส่ง", search ด้วย PO ID/ข้อมูลลูกค้า) → ระบบสร้าง DN ราย PO → รวมเป็นรอบ (คนขับ/เบอร์/route/ประเภทรถ) · DN Reject→PO กลับพร้อมจัดส่ง+raise Sale · Postpone→PO พร้อมจัดส่ง+flag+วันที่ ค้างคิว · รอบจบเมื่อทุก DN ถึงสถานะสุดท้าย

โผล่ใน: UC6 (เดี่ยว), UC7 (รวมสำเร็จหมด), UC8 (Partially)

3. User Stories + Acceptance Criteria

US-SHP-01 — สร้างรอบจัดส่ง + DN ราย PO Must

As a ฝ่ายจัดส่ง, I want สร้างรอบและเลือก PO พร้อมจัดส่ง (ค้นด้วย PO ID/ข้อมูลลูกค้า), so that ได้ DN ราย PO ให้ลูกค้าเซ็น

Happy Given PO-176 สถานะ "พร้อมจัดส่ง" When สร้างรอบ SHP-20260708-0044 เลือก PO-176 + กรอกคนขับ/เบอร์/route/ประเภทรถ Then gen DN-20260708-00119 (=PO-176); รอบ=รับเข้ารอบ→In-Route; PO→กำลังจัดส่ง (C9)
Edge — search by customer + หลาย PO Given ค้น PO ด้วยชื่อบริษัท/เบอร์ contact When เลือกหลาย PO พร้อมจัดส่งเข้ารอบเดียว Then สร้าง 1 DN ต่อ 1 PO (หลาย DN ในรอบเดียว)
Error — เลือก PO ที่ไม่พร้อมจัดส่ง Given ค้นเจอ PO ที่ยังผลิตอยู่ When พยายามเพิ่มเข้ารอบ Then เลือกไม่ได้ (เฉพาะ "พร้อมจัดส่ง") + แจ้งเหตุผล

US-SHP-02 — DN Delivered / Rejected / Postponed Must

As a ฝ่ายจัดส่ง, I want อัปเดตผลการส่งราย DN, so that สถานะ PO และคิวถูกต้อง

Happy — Delivered Given DN-00119 กำลังนำส่ง When ลูกค้ารับ+เซ็น → กด "ส่งถึงแล้ว" Then DN=ส่งถึงแล้ว; PO=ส่งถึงแล้ว; เริ่มนับ overdue; ยิง Noti → Finance+Sale (C10)
Edge — Rejected Given DN-00123 (PO-175) ลูกค้าปฏิเสธ (ผิดสเปก) When กด "ถูกปฏิเสธ" Then PO-175→พร้อมจัดส่ง (กลับคิว) + raise Sale; รอสร้าง DN รอบใหม่ (C11)
Error/Edge — Postponed Given DN-00124 (PO-178) ลูกค้าขอเลื่อนเป็น 10/07 When กด "เลื่อนส่ง" + ระบุวันที่ Then PO-178→พร้อมจัดส่ง + flag "กันจัดส่ง Postpone 10/07" ค้างคิว; ยิง Noti → Shipping (C12)

US-SHP-03 — Reconcile รอบ (จบรอบ / Partially) + print DN Must

As a ฝ่ายจัดส่ง, I want เห็นสถานะรอบ reconcile จาก DN และพิมพ์ DN ทีละใบ, so that ปิดรอบถูกต้องและมีเอกสารให้ลูกค้าเซ็น

Happy — จบรอบ Given SHP-0045 มี 2 DN ส่งถึงครบ When DN ทุกใบถึงสถานะสุดท้าย Then รอบ=จบรอบ (Closed) auto (ref)
Edge — Partially Given SHP-0046: 1 ส่งถึง + 1 ปฏิเสธ + 1 เลื่อน When ดูหัวรอบ Then แสดง "ส่งบางส่วน (Partially)" + breakdown 3 DN; รอบยังไม่ปิดจนทุก DN ถึงสถานะสุดท้าย (Rejected/Postponed = จัดการแล้ว)
Error — print DN ที่ยังไม่สร้าง Given PO ยังไม่ถูกใส่รอบ (ยังไม่มี DN) When พยายามพิมพ์ DN Then ไม่มีปุ่มพิมพ์ / error "ยังไม่มีใบจัดส่งสำหรับ PO นี้"
