Module: QC (ควบคุมคุณภาพ) Should

2 งาน: ตรวจรับ Lot วัตถุดิบขาเข้า + ตรวจ Batch การผลิต · การตัดสิน QC ทำที่หน้านี้เท่านั้น (หน้าผลิตไม่มีปุ่มตัดสิน)

1. ภาพรวม + Use cases

QC ตัดสิน Batch ผ่าน/ไม่ผ่านราย line · ไม่ผ่าน → PRD Rework + feedback บังคับ + gen Batch run ถัดไป · ตรวจรับ Lot ขาเข้า ผ่าน→Lot พร้อมใช้ (อาจปิด PR) / ไม่ผ่าน→Lot ระงับ→คืนของ · เห็น GMP chain (Lot→Batch→PO)

โผล่ใน: UC3, UC4 (rework PO-181), UC5

3. User Stories + Acceptance Criteria

US-QC-01 — ตรวจ Batch (ผ่าน) Must

As a เจ้าหน้าที่ QC, I want ตัดสิน Batch ราย line ว่าผ่าน, so that ปล่อยงานไปสู่การจัดส่ง

Happy Given B-PO-202607-000176-1-1 สถานะ "รอ QC" When QC เลือก "✓ ผ่าน" Then Batch=QC ผ่าน; PRD line=พร้อมส่งมอบ; ถ้าทุก PRD ของ PO ผ่าน → PO=พร้อมจัดส่ง + ยิง Noti → Shipping (C5)
Edge — บาง line ยังไม่ผ่าน Given PO 2 line, ผ่านแค่ line 1 When ตัดสิน line 1 ผ่าน Then PO ยังไม่ พร้อมจัดส่ง (แสดง "ผ่าน 1/2")
Error — ตัดสิน Batch ที่ไม่อยู่คิว Given Batch ยังไม่ถูกส่งตรวจ (ไม่อยู่สถานะรอ QC) When พยายามตัดสิน Then ไม่มีในคิว QC / error "รายการไม่พร้อมตรวจ"

US-QC-02 — ตรวจ Batch (ไม่ผ่าน → Rework) Must

As a QC, I want ตัดสินไม่ผ่านพร้อม feedback, so that ฝ่ายผลิตผลิตซ้ำเฉพาะ line ที่เสีย

Happy Given B-PO-202607-000181-2-1 (โฟม) รอ QC When QC เลือก "✕ ไม่ผ่าน" + feedback "เนื้อโฟมเป็นก้อน ปั๊มไม่ออก" Then Batch=ไม่ผ่าน; PRD line 2=Rework; ยิง Noti → Production; trace บันทึก feedback (C6)
Edge — เห็นประวัติ run Given line เคยไม่ผ่าน run1 แล้วผลิต run2 When เปิดคิว QC ของ run2 Then เห็น run2 เป็นรายการใหม่ + ประวัติ run1 ไม่ผ่าน + feedback เดิม
Error — ไม่ผ่านโดยไม่มี feedback Given ช่อง feedback ว่าง When กด "ไม่ผ่าน" Then ไม่บันทึก + error "ต้องระบุ feedback" (บังคับ)

US-QC-03 — ตรวจรับ Lot วัตถุดิบขาเข้า Should

As a QC, I want ตรวจรับ Lot ขาเข้า, so that เฉพาะวัตถุดิบผ่านคุณภาพเท่านั้นถูกใช้ผลิต

Happy Given Lot L-GLY-2607 สถานะ "รอตรวจรับ" When QC ตัดสิน "ผ่าน" Then Lot=พร้อมใช้ผลิต; ถ้าอ้าง PR อาจปิด PR; ยิง Noti → Stock (C18)
Edge — ไม่ผ่าน → คืนของ Given Lot ตรวจเจอเสียหาย When QC ตัดสิน "ไม่ผ่าน" Then Lot=ระงับ → เปิดใบคืนของ supplier (ดู Return)
Error — ตัดสิน Lot ที่ผ่านแล้ว Given Lot=พร้อมใช้ผลิตแล้ว When พยายามตัดสินซ้ำ Then ไม่อยู่คิว / error "Lot นี้ตรวจรับแล้ว"
