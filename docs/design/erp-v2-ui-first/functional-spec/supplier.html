Module: Supplier Should

เพิ่ม/แก้ + ผูกวัตถุดิบรายตัว + ราคารับซื้อ (price matrix supplier×วัตถุดิบ) + Active/Inactive · ไม่มีฟังก์ชันรับเข้าคลังในหน้านี้ · เลข SUP-{NN} · Lot prefix ต่อ supplier

1. ภาพรวม + Use cases

ฝ่ายจัดซื้อ/คลังจัดการ supplier · ราคารับซื้อต่อ supplier ต่อวัตถุดิบ (feed BOM cost = max active supplier) · inactive → ถูกตัดจากการคิด BOM cost + ห้ามซื้อใหม่

Dataset: SUP-01/02 Active, SUP-03 อโรมา Inactive (มีน้ำหอม → BOM-02 ต้อง override)

3. User Stories + Acceptance Criteria

US-SUP-01 — เพิ่ม/แก้ supplier + price matrix Must

As a ฝ่ายจัดซื้อ, I want ผูกวัตถุดิบและราคารับซื้อต่อ supplier, so that ราคาทุน BOM คำนวณได้และรับซื้อถูกราคา

Happy Given สร้าง SUP-01 "กลีเซอรีนไทย" Lot prefix L-GLY- When ค้นและผูกวัตถุดิบ "กลีเซอรีน" ราคา 45, "น้ำหอมกุหลาบ" 900 Then บันทึก price matrix (supplier×วัตถุดิบ); ราคานี้ใช้คิด BOM cost (max active)
Edge — วัตถุดิบหลาย supplier Given กลีเซอรีนมีทั้ง SUP-01 (45) และ supplier อื่น When ผูกราคาต่างกัน Then เก็บหลายรายการ (1 วัตถุดิบ N supplier); BOM ใช้ราคาสูงสุดของ active
Error — ราคา/วัตถุดิบซ้ำ Given ผูกวัตถุดิบเดิมซ้ำใน supplier เดียว หรือราคาติดลบ When บันทึก Then error "วัตถุดิบผูกแล้ว" / "ราคาต้อง ≥ 0"

US-SUP-02 — Active/Inactive Must

As a ฝ่ายจัดซื้อ, I want ตั้ง supplier เป็น inactive, so that หยุดซื้อและตัดออกจากการคิดราคาทุนโดยไม่ลบประวัติ

Happy Given SUP-03 อโรมา When ตั้ง Inactive Then SUP-03 ถูกตัดจากการคิด BOM cost (max active); หายจาก dropdown ซื้อใหม่; Lot เดิมยังใช้ผลิตต่อได้
Edge — วัตถุดิบเหลือ supplier เดียวที่ inactive Given น้ำหอมมีเฉพาะ SUP-03 (inactive) When คิด BOM-02 cost Then ไม่มีราคา active → BOM บล็อกจนกรอก override (ดู BOM)
Error — รับเข้าคลังในหน้านี้ Given ผู้ใช้มองหาฟังก์ชันรับเข้า When อยู่หน้า supplier Then ไม่มีฟอร์มรับเข้า (ย้ายไป Stock/Goods Receipt)
