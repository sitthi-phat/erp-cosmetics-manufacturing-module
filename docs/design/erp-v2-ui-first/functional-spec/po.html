Module: PO — ใบสั่งซื้อ (Purchase Order) Tier 1

ราง Fulfilment: ร่าง→ยืนยันแล้ว→กำลังผลิต→พร้อมจัดส่ง→กำลังจัดส่ง→ส่งถึงแล้ว (+ยกเลิก/เปิดใหม่) · ราง Billing แยก (ดู Invoice) · เลข PO-{YYYYMM}-{NNNNNN}

Stock Reservation (ปอนด์ล็อก 2026-07-10): ยืนยัน PO = จองวัตถุดิบ (Reserve) = ΣBOM×qty ต่อ line (Available = on_hand − Reserved ลดลง; ยังไม่ตัด on_hand จริง) → ตัดจริง (Consume) ตอน "เริ่มผลิต" ราย Batch (convert จอง→ตัด FIFO — ดู US-PRD-02) · Cancel ก่อนเริ่มผลิต = คืนจองอัตโนมัติ · Cancel หลังเริ่มผลิต = dialog ถามคืน/ไม่คืนวัตถุดิบที่ตัดไปแล้ว · ที่มา: stock-reservation.md

1. ภาพรวม + Use cases

หน้าเปิดใบสั่งซื้อของฝ่ายขาย ขายได้ทั้ง สินค้า BOM และวัตถุดิบตรง · ราคา default จากราคาขาย BOM แต่แก้ได้เสมอ (ราคา 0 ได้) · วัตถุดิบขาด (Available ไม่พอ) = เตือน ไม่บล็อก + auto Purchase Request · ยืนยันแล้วแต่ละ line เข้าคิวฝ่ายผลิต "รอรับงาน" (ยังไม่เกิด PRD) พร้อมจองวัตถุดิบ

โผล่ใน use case: UC1 (Draft PO-191), UC2 (Hold PO-188), UC4 (rework PO-181), UC5/6/8

3. User Stories + Acceptance Criteria

US-PO-01 — สร้าง PO ร่าง Must

As a พนักงานขาย (Sale), I want เปิดใบสั่งซื้อร่างพร้อมเลือกสินค้า BOM/วัตถุดิบ แก้จำนวนและราคา/หน่วยได้, so that รับออเดอร์ลูกค้าได้เร็วโดยไม่ติดข้อจำกัดราคา

Happy Given Sale อยู่หน้า po-create เลือกลูกค้า "ร้านสวยใส" และเพิ่มบรรทัด "เซรั่มวิตซี 15% (BOM-01) × 100" When กด "บันทึกร่าง" Then ระบบสร้าง PO-202607-000191 สถานะ = "ร่าง (Draft)", ราง billing = "ยังไม่วางบิล", ราคา/หน่วย default = ราคาขาย BOM (220), มี trace "สร้างโดย อารดา + เวลา" — ร่างยังไม่จองวัตถุดิบ (จองตอนยืนยัน)
Edge — ราคา 0 / วัตถุดิบขาด Given Sale แก้ราคา/หน่วยของบรรทัดเป็น 0 และเพิ่มสินค้าที่ BOM ต้องใช้วัตถุดิบเกิน Available When กดบันทึกร่าง/ยืนยัน Then ระบบ ยอมรับราคา 0 (ไม่บล็อก) และแสดง warning "วัตถุดิบอาจไม่พอ" แบบไม่บล็อก พร้อมเสนอสร้าง Purchase Request เฉพาะส่วนที่ขาด — PO เดินหน้าได้ปกติ
Error — ไม่มีลูกค้า/ไม่มีบรรทัด Given Sale ยังไม่เลือกลูกค้า หรือไม่มีบรรทัดสินค้าเลย When กดบันทึก Then ระบบไม่บันทึก แสดง error "ต้องเลือกลูกค้า" และ/หรือ "ต้องมีอย่างน้อย 1 รายการ" ที่ field ที่เกี่ยวข้อง

US-PO-02 — ยืนยัน PO → เข้าคิวผลิต + จองวัตถุดิบ Must

As a Sale, I want ยืนยันใบสั่งซื้อและให้ระบบจองวัตถุดิบให้อัตโนมัติ, so that งานถูกส่งเข้าคิวผลิต วัตถุดิบถูกกันไว้ (ไม่โดนออเดอร์อื่นแย่ง) และลูกค้าใหม่ถูกยกระดับเป็นลูกค้าประจำ

Happy — Confirmed + reservation Given PO-202607-000191 สถานะ "ร่าง" ของลูกค้า "ผู้สนใจ (Lead)"; BOM-01 ใช้ วิตซี 0.02, กลีเซอรีน 0.01 ต่อหน่วย × 100 When Sale กด "ยืนยัน PO" Then PO = "ยืนยันแล้ว (Confirmed)"; แต่ละ line เข้าคิว "รอรับงาน" (ยังไม่ออกเลข PRD); สร้าง reservation ต่อ line = ΣBOM×qty (วิตซี 2, กลีเซอรีน 1) → Reserved เพิ่ม / Available ลด (on_hand ไม่เปลี่ยน); ลูกค้า Lead→Active (auto ใบแรก); ยิง Notification → ฝ่ายผลิต — ดู Cascade C1
Edge — จองเกิน Available (ไม่บล็อก) / ลูกค้า Follow-up-Inactive Given วัตถุดิบของ line มี Available น้อยกว่าที่ต้องจอง (เช่นเหลือ 1 ต้องจอง 2) หรือลูกค้าเป็น "ต้องติดตาม"/"ห่างหาย" When ยืนยัน PO Then อนุญาตจอง จน Available ติดลบ + เตือน "จองเกินของในคลัง X หน่วย (รอรับเข้า)" (ไม่บล็อก, สอดคล้อง warning-not-block); ลูกค้ากลับเป็น "ลูกค้าประจำ (Active)" อัตโนมัติเมื่อมี order ใหม่
Error — ลูกค้า Blacklist/Disabled Given ลูกค้าเป็น "บัญชีดำ (Blacklist)" หรือ "ปิดใช้งาน (Disabled)" When Sale พยายามยืนยัน PO Then ระบบเตือน/บล็อกการยืนยัน (ไม่สร้าง reservation) และแสดงเหตุผลจาก comment ที่บันทึกไว้

US-PO-03 — ค้นหา PO 3 แบบวันที่ Should

As a Sale/ผู้ดูภาพรวม, I want ค้นหา PO ด้วยวันที่สร้าง / วันที่จัดส่งจริง / วันที่ต้องการรับสินค้า, so that หาใบที่ต้องติดตามได้เร็ว

Happy Given po-list มี PO 11 ใบใน dataset When เลือกประเภทวันที่ = "วันที่สร้าง" ช่วง 02–05/07 แล้วค้นหา Then แสดงเฉพาะ PO ที่สร้างในช่วงนั้น เรียงใหม่, มี pagination เมื่อเกิน 1 หน้า
Edge — ไม่พบผล Given เลือกช่วงวันที่ที่ไม่มี PO When ค้นหา Then แสดง empty state "ไม่พบใบสั่งซื้อในช่วงที่เลือก" (ไม่ error) + ปุ่มล้างตัวกรอง
Error — ช่วงวันที่กลับด้าน Given วันที่เริ่ม > วันที่สิ้นสุด When ค้นหา Then แสดง error "วันที่เริ่มต้องไม่เกินวันที่สิ้นสุด" ไม่ยิง query

US-PO-04 — ยกเลิก PO + เปิดใหม่ (คืนจอง / dialog คืนวัตถุดิบที่ตัดแล้ว) Must

As a Sale/Admin, I want ยกเลิก PO ได้ทุกขั้น โดยระบบจัดการวัตถุดิบที่จอง/ตัดไปแล้วให้ถูกต้อง และเปิดใหม่คงเลขเดิม, so that ยกเลิกได้โดย stock ไม่เพี้ยนและเลขเอกสารไม่กระโดด (gapless)

Happy — cancel ก่อนเริ่มผลิต = คืนจองอัตโนมัติ Given PO ที่ยืนยันแล้วแต่ ยังไม่มี line ใดเริ่มผลิต (มีแต่ reservation ยังไม่ consume) When กด "ยกเลิก PO" + กรอกเหตุผล (บังคับ) Then PO = "ยกเลิก (Cancelled)"; release reservation ที่ยังไม่ consume ทั้งหมดอัตโนมัติ (Reserved↓, Available↑, on_hand ไม่แตะ); คิว/PRD ที่เกิดถูกยกเลิกตาม + trace (C19); "เปิดใหม่ (Cancelled→Draft)" คงเลข PO เดิม และ re-confirm = จองใหม่
Edge — cancel หลังเริ่มผลิต = dialog คืน/ไม่คืน (ปอนด์ล็อก) Given PO ที่มีบาง line เริ่มผลิตแล้ว (Batch ตัด on_hand จริงไปแล้ว) When กด "ยกเลิก PO" Then ระบบเปิด dialog "คืนวัตถุดิบที่ตัดไปแล้วเข้า stock หรือไม่?" + เหตุผลบังคับ → เลือก "คืน" = สร้าง stock movement บวก on_hand กลับ (ตาม lot ที่ตัด) + trace; เลือก "ไม่คืน" = on_hand คงถูกตัด (ของถูกใช้/เสียไปแล้ว) + trace เหตุผล · reservation ที่ยังไม่ consume = release อัตโนมัติทั้งสองทาง
Error — ไม่ใส่เหตุผล / ไม่มีสิทธิ์ Given ผู้ใช้กดยกเลิก/ปิด dialog โดยเว้นเหตุผล หรือไม่มีสิทธิ์ D/Admin ของ module PO When ยืนยัน Then ระบบไม่ดำเนินการ (ไม่แตะ reservation/stock) แสดง error "ต้องระบุเหตุผล" หรือ "ไม่มีสิทธิ์ยกเลิก/เปิดใหม่"

US-PO-05 — แก้ไข PO (เคส Hold) + ปรับจองตาม delta Must

As a Sale, I want แก้ไข PO ได้ทุกช่อง (จำนวน/สินค้า/ราคา/วันส่ง) เมื่องานถูก Hold และให้ระบบปรับยอดจองตามที่แก้, so that แก้ตามลูกค้าแล้ววัตถุดิบที่กันไว้ตรงกับความต้องการใหม่ พร้อมร่องรอยตรวจสอบ

Happy — แก้จำนวน → ปรับ reservation delta Given PO-202607-000188 มี PRD ถูก Hold; line เดิมจองวิตซีไว้ 2.4 (120 หน่วย) When Sale แก้จำนวนจาก 120 → 150 แล้วบันทึก Then ระบบ recompute ΣBOM×qty ใหม่ → reserve เพิ่ม delta (วิตซี +0.6); ถ้าแก้ลดจำนวน = release ส่วนเกิน; ทุกช่องที่แก้มี trace ราย field (ใคร/จาก→เป็น/เมื่อไหร่); PRD กลับสู่สายผลิตต่อได้
Edge — ตั้งลูกค้า Follow-up Given Hold เหตุ "รอลูกค้ายืนยันสูตร" When Sale ตั้งลูกค้าเป็น "ต้องติดตาม" + comment Then ลูกค้า → Follow-up, โผล่ tile "ต้องติดตาม" ใน Sale Dashboard (ดู Cascade C8); การปรับจองเป็นอิสระจากการตั้ง Follow-up
Error — แก้โดยไม่มีสิทธิ์ Update Given ผู้ใช้ไม่มีสิทธิ์ Update ของ module PO When พยายามแก้ค่าใน po-detail Then ช่องแก้ไขถูก disable/ปุ่มบันทึกซ่อน + ข้อความ "อ่านอย่างเดียว — ไม่มีสิทธิ์แก้ไข" (ไม่แตะ reservation)

US-PO-06 — สถานะ 2 ราง reconcile + force override Should

As a ผู้ดู PO, I want เห็นสถานะราง Fulfilment และ Billing แยกกันและตรงกับความจริงข้าม module, so that รู้สถานะจริงของงานทุกด้าน; Admin แทรกสถานะ (override) ได้เมื่อจำเป็น

Happy Given PO-202607-000176 ผลิตครบ ผ่าน QC และส่งถึงแล้ว When เปิด po-detail Then ราง Fulfilment = "ส่งถึงแล้ว", ราง Billing = "รอชำระ (INV-2026-000135)" — สองรางแสดงแยกและตรงกับหน้า production/qc/shipping/invoice
Edge — force override (Admin) Given ผู้ใช้มีสิทธิ์ Admin ของ module PO When ใช้การ์ด "เปลี่ยนสถานะ PO" ข้ามลำดับ + เหตุผล Then เปลี่ยนได้ + trace ระบุ "force override โดย <user> เหตุผล …"; ผู้ที่ไม่ใช่ Admin ไม่เห็นตัวเลือกข้ามลำดับ
Error — override โดยไม่มีสิทธิ์ Given ผู้ใช้ไม่มีสิทธิ์ Admin When พยายามข้ามลำดับสถานะ Then ไม่มีตัวเลือกให้ / ถ้าเรียก API ตรงต้องถูกปฏิเสธด้วย 403 + ไม่มีการเปลี่ยนสถานะ

US-PO-07 — Suggest + เช็ควัตถุดิบ (เทียบ Available) + ราคา default ตอนสร้าง Must

As a Sale, I want ระบบ suggest สินค้าและคำนวณว่าวัตถุดิบ "ใช้ได้ (Available)" พอไหมพร้อมเติมราคา default ให้อัตโนมัติ, so that เปิด PO ได้เร็วและเห็นความเสี่ยงวัตถุดิบตามยอดที่ยังไม่ถูกจองก่อนยืนยัน

สูตรเช็ควัตถุดิบพอ (ราย component): ต้องใช้ = Σ ต่อ line (BOM component qty × จำนวนสั่ง) · เทียบกับ Available = on_hand − Reserved (ไม่ใช่ on_hand ล้วน) · ถ้า ต้องใช้ > Available → "ไม่พอ" ส่วนที่ขาด = ต้องใช้ − Available · ราคา default: line BOM = ราคาขายของ BOM · line วัตถุดิบตรง = default_sell_price ของวัตถุดิบ · แก้ได้ (0 ได้)
Happy — suggest + auto price Given Sale พิมพ์ค้นในช่องสินค้า po-create When เลือก "เซรั่มวิตซี 15% (BOM-01)" จาก dropdown แบบค้นได้ และใส่จำนวน 100 Then บรรทัดเติม ราคา/หน่วย default = ราคาขาย BOM-01 (220) อัตโนมัติ; ถ้าเลือกวัตถุดิบตรงจะเติม default_sell_price แทน
Edge — เช็ค Available (on_hand − Reserved) Given BOM-01 ใช้ วิตซี 0.15 กก./หน่วย; on_hand วิตซี = 12 กก. แต่ถูกจองไปแล้ว 5 กก. → Available = 7 และสั่ง 100 หน่วย When ระบบคำนวณตอนเพิ่มบรรทัด Then ต้องใช้ = 0.15×100 = 15 > Available 7 → แสดง warning (ไม่บล็อก) "วิตซีอาจไม่พอ: ต้องใช้ 15 · ใช้ได้ 7 · ขาด 8 กก." + เสนอสร้าง PR เฉพาะส่วนที่ขาด (ดู C15)
Error — จำนวน ≤ 0 / BOM ไม่มีต้นทุน Given ใส่จำนวนเป็น 0/ติดลบ หรือเลือก BOM ที่ยัง block เพราะไม่มี supplier active (ดู BOM) When เพิ่มบรรทัด/บันทึก Then error "จำนวนต้องมากกว่า 0" ที่บรรทัดนั้น; กรณี BOM ยังไม่มีต้นทุน แจ้ง "สูตรนี้ยังคิดราคาทุนไม่ได้ ให้แก้ที่ BOM ก่อน"

4. Data rules / สถานะ Entity

รายการกติกา (อ้าง entity-status-map.md §1.2/§1.3 + stock-reservation.md)
เลข POPO-{YYYYMM}-{NNNNNN} gapless ต่อเดือน · reopen คงเลขเดิม
สถานะ Fulfilmentร่าง → ยืนยันแล้ว → กำลังผลิต → พร้อมจัดส่ง → กำลังจัดส่ง → ส่งถึงแล้ว · ยกเลิก↔ร่าง
สถานะ Billingยังไม่วางบิล → วางบิลแล้ว → ชำระแล้ว / เกินกำหนด (ดู Invoice)
บรรทัดสินค้าBOM หรือวัตถุดิบตรง · จำนวน > 0 · ราคา/หน่วย ≥ 0 (0 ได้) · 1 line = 1 PRD (เมื่อรับงาน)
ราคา defaultline BOM = ราคาขาย BOM · line วัตถุดิบตรง = default_sell_price · แก้ได้เสมอ
Reservationยืนยัน PO → จอง = ΣBOM×qty ต่อ line (Reserved↑, Available↓, on_hand ไม่แตะ) · เช็คตอน suggest เทียบ Available · จองเกิน=เตือนไม่บล็อก · Hold edit=ปรับ delta · consume ตอนเริ่มผลิต (US-PRD-02)
Cancelก่อนเริ่มผลิต=release จองอัตโนมัติ · หลังเริ่มผลิต=dialog คืน/ไม่คืนวัตถุดิบที่ตัดแล้ว + เหตุผลบังคับ; reservation ที่ยังไม่ consume=release เสมอ
บังคับ commentยกเลิก (+ตัวเลือก dialog), force override, แก้ไขเคส Hold (trace ราย field)
สิทธิ์ (RUCDAA)สร้าง=C · แก้=U · ยกเลิก/เปิดใหม่=D · force override=Admin (ดู RUCDAA)

6. ความต่อเนื่องข้าม module

  • ยืนยัน PO → คิวผลิต "รอรับงาน" + จองวัตถุดิบ + Lead→Active (C1)
