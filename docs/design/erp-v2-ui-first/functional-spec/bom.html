Module: BOM — สูตรการผลิต Should

ราคาทุน = ราคารับซื้อ สูงสุดของ supplier ที่ active + user override ได้ + snapshot ตอนบันทึก + badge เตือน "ราคาทุนอาจล้าสมัย" · ราคาขาย mandatory · เลข BOM-{NN}

1. ภาพรวม + Use cases

สร้างสูตรเต็มรูปแบบ เลือกวัตถุดิบประกอบ (ไม่จำเป็นต้องมี stock แต่โชว์ stock) · ราคาทุนคิดจาก max active supplier ณ ตอนสร้าง/แก้ + snapshot · ถ้าไม่มี supplier active → บล็อกจนกรอก override

Dataset: BOM-01/03/04 (snapshot), BOM-02 ครีมกันแดด (น้ำหอมจาก SUP-03 inactive → override 12.40)

3. User Stories + Acceptance Criteria

US-BOM-01 — สร้างสูตร + คิดราคาทุน (max active) + snapshot Must

As a ฝ่ายผลิต/สูตร, I want สร้างสูตรและให้ระบบคิดราคาทุนจาก supplier active สูงสุด แล้ว snapshot, so that ต้นทุนคงที่จนกว่าจะแก้สูตรใหม่

Happy Given BOM-01 ใช้วิตามินซี+กลีเซอรีน มี supplier active When บันทึกสูตร Then ราคาทุน = ผลรวม (max active supplier price × ปริมาณ) = 18.00; snapshot ไว้; ราคาขาย mandatory = 220
Edge — badge ล้าสมัย Given ราคา supplier active ปัจจุบันต่างจาก snapshot When เปิดดู BOM Then แสดง badge "ราคาทุนอาจล้าสมัย"; ราคาทุนยังใช้ snapshot จนกว่า save ใหม่
Error — ไม่กรอกราคาขาย Given ราคาขายว่าง When บันทึกสูตร Then error "ราคาขายจำเป็น (mandatory)" — ไม่บันทึก

US-BOM-02 — ไม่มี supplier active → บล็อกจนกรอก override Must

As a ฝ่ายสูตร, I want ถ้าวัตถุดิบไม่มี supplier active ให้ระบบบังคับกรอกราคาทุน override, so that ราคาทุนไม่เป็น 0 โดยไม่ตั้งใจ

Happy — override Given BOM-02 มีน้ำหอม (SUP-03 inactive เท่านั้น) When ผู้ใช้กรอกราคาทุน override 12.40 Then บันทึกได้ด้วยราคา override + snapshot
Edge — บาง component มี active Given บาง component มี active บาง component ไม่มี When คิดราคาทุน Then component ที่มี active คิดอัตโนมัติ, component ที่ไม่มีต้อง override
Error — บันทึกโดยไม่ override Given มี component ไม่มี supplier active และยังไม่กรอก override When บันทึกสูตร Then ระบบบล็อก + error "วัตถุดิบไม่มีราคา supplier active กรุณากรอกราคาทุน"
