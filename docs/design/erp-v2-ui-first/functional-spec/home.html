1. ภาพรวม + Use cases

3. User Stories + Acceptance Criteria

US-HOME-01 — งานที่รอ (Task inbox ต่อ role) + deep link Must

As a ผู้ใช้ (ตาม role), I want เห็นงานค้างที่รอผมทำอยู่ในหน้าแรก และกดไปทำต่อได้ทันที, so that ไม่พลาดงานและเริ่มทำได้เร็วโดยไม่ต้องไล่หาเอง

Happy Given สมชาย (ฝ่ายขาย/CS) login โดยมี PO-202607-000175 (ถูกปฏิเสธ), PO-178 (Postpone 10/07), PO-176 (รอชำระ INV-135) When เปิดหน้า Home แล้วกด "เปิด" ที่แถว PO-175 Then การ์ด "งานที่รอคุณอยู่" แสดง 3 งานพร้อม badge สถานะ ("ถูกปฏิเสธ → ต้องติดต่อลูกค้า" / "Postpone 10/07" / "รอชำระ · ครบ 07/08"); กด "เปิด" = deep link ไป po-detail.html ของ PO นั้น
Edge — งานตาม role ต่างกัน + นับตรง source เดียว Given user เป็นฝ่ายผลิต (มีสิทธิ์ Read Production) หรือการเงิน (Read Finance) When เปิด Home Then ฝ่ายผลิตเห็น "งานรอรับ (รอรับงาน)"/Rework, การเงินเห็น "รอชำระ/เกินกำหนด" — เฉพาะงานของ role ตนเอง; จำนวนงานค้างตรงกับ dashboard tile และ notification ของ user คนนั้น (ข้อมูล source เดียวกัน)
Error — งานที่ไม่มีสิทธิ์ Read Given user ไม่มีสิทธิ์ Read ของ module ที่งานสังกัด When โหลด task inbox Then งานนั้นไม่แสดงในหน้า Home (task inbox scope ตามสิทธิ์ Read); กด deep link ตรงไปหน้าที่ไม่มีสิทธิ์ → guard 403

US-HOME-02 — Onboarding ผู้ใช้ใหม่ + เมนูตามสิทธิ์ Should

As a พนักงานที่ไม่เคยใช้ ERP, I want คำแนะนำเริ่มต้นและเห็นเฉพาะเมนู/งานที่ทำได้, so that ใช้งานเองได้โดยไม่มีคนสอนและไม่กดผิด

Happy Given user login เข้า Home When ดูหน้าแรก Then แสดงกล่องแนะนำ 💡 "ยังใหม่กับระบบใช่ไหม? การ์ดด้านล่างพาคุณไปงานหลักทีละขั้น…" + การ์ด "ต้องรู้ก่อนเริ่ม" (อธิบาย Lot = ล็อตวัตถุดิบ vs Batch = รอบผลิต, สถานะภาษาไทยเสมอ, ทุกการเปลี่ยนสถานะมี trace)
Edge — เมนู/quick action ตามสิทธิ์ Given role ที่ไม่มีสิทธิ์ Read บาง module When ดูเมนูซ้าย + การ์ด "งานที่ทำบ่อย" Then แสดงเฉพาะ module/ทางลัดที่ role มีสิทธิ์ Read (ผู้ใช้ไม่เห็นเมนูที่ทำไม่ได้)
Error — ไม่แสดง enum/รหัสดิบ Given เนื้อหา onboarding + task inbox When render Then ทุกสถานะเป็นป้ายภาษาไทยอ่านออก (เช่น "รอ QC", "บัญชีดำ") — ไม่มี enum/รหัสดิบโผล่

US-HOME-03 — Quick actions (งานที่ทำบ่อย) + empty state Should

As a ผู้ใช้ routine, I want ทางลัดงานที่ทำบ่อยและหน้าโล่งที่ไม่น่ากลัวเมื่อไม่มีงานค้าง, so that เริ่มงานได้ในคลิกเดียวและรู้ว่าไม่มีงานตกค้าง

Happy Given สมชาย (ฝ่ายขาย) อยู่ Home When กดการ์ด "เปิดใบสั่งซื้อใหม่" Then แสดง quick actions 3 อัน (เปิด PO→po-create.html, ลูกค้าของฉัน→customers.html, ติดตาม PO→po-list.html); คลิก = ไปหน้านั้นทันที (ลดคลิก)
Edge — empty state (ไม่มีงานค้าง) Given user ที่ไม่มีงานค้างเลย When เปิด Home Then การ์ด "งานที่รอคุณอยู่" แสดง empty state "ไม่มีงานค้าง 🎉" (ไม่ error) + ยังเห็น quick actions + onboarding ปกติ
Error — โหลดข้อมูลล้มเหลว Given API dashboard/noti ตอบผิดพลาดชั่วคราว When เปิด Home Then แสดงข้อความ "โหลดงานที่รอไม่สำเร็จ ลองใหม่" + ปุ่มลองใหม่ (ไม่หน้าขาว/ค้าง) · quick actions/onboarding ยังใช้ได้

4. Data rules / สถานะ Entity

รายการกติกา
Task inboxรวมงานค้างของ user ตาม role (scope = สิทธิ์ Read) · ไม่มี entity ใหม่ — รวมจากสถานะ PO/PRD/DN/Invoice ที่เกี่ยวกับ user
แหล่งข้อมูล/การนับsource เดียวกับ dashboard aggregates + notification (ไม่คำนวณซ้ำ ต้องได้ตัวเลขตรงกัน)
deep linkทุกงาน "เปิด" → หน้ารายละเอียดจริง (po-detail/production/invoice…) พร้อม context
onboardingข้อความคงที่ + Lot vs Batch glossary (ดู Glossary) · ไม่มี enum ดิบ
สิทธิ์แสดงงาน/เมนู/quick action ตาม Read permission ราย module (ดู RUCDAA)

5. API / Database (Tech-Lead)

API / DB (Tech-Lead): api-dashboard.html ↗ (aggregates/task list ต่อ role) · api-notification.html ↗ (inbox ราย user) · db-schema.html ↗ — ไม่ต้องมี API ใหม่: Home reuse dashboard aggregates (scope ตาม Read) + notification inbox; ต้องให้ตัวเลข task inbox = dashboard tile = noti badge (source เดียวกัน). ถึง TL: ยืนยันว่า api-dashboard มี endpoint "my-tasks/per-role task list" ที่ Home เรียกใช้ได้ ถ้ายังไม่มีให้เพิ่ม 1 endpoint รวมงานค้างของ user (ไม่ใช่ per-module tile)

6. ความต่อเนื่องข้าม module

  • ทุกเหตุการณ์ที่ยิง notification (Noti matrix) → โผล่ใน task inbox ของ role ปลายทางด้วย (เช่น DN Rejected→งานฝ่ายขาย, PO Confirmed→งานฝ่ายผลิต, Overdue→งานการเงิน)
  • นับตรงกับ Dashboard tile + Notification ของ user คนเดียวกัน
