Cross-cutting: List/Table Page Conventions Must

มาตรฐาน search / filter fields · sort · default sort · page-size · empty state ต่อทุกหน้า list — เพื่อ Engineer/QA ไม่ต้องเดา · ตัวกรองทุกตัวมาจาก field/สถานะที่นิยามไว้แล้วใน entity-status-map + Glossary (ไม่มีการประดิษฐ์ field ใหม่)

มาตรฐานร่วมทุกหน้า list: page-size default = 20/หน้า (เลือก 10/20/50) · pagination "แสดง X–Y จาก Z" + เลขหน้า · คงตัวกรอง+หน้าเมื่อ refresh/กลับมา (ไม่รีเซ็ต) · ไม่พบผล = empty state ข้อความเฉพาะหน้า + ปุ่มล้างตัวกรอง · ทุกสถานะแสดงเป็นป้ายไทย (ไม่โชว์ enum ดิบ) · ตัวกรองที่ผู้ใช้ไม่มีสิทธิ์ Read เห็นข้อมูลข้าม module ถูก scope ตามสิทธิ์

2. User Story + Acceptance Criteria

US-LST-01 — ตัวกรอง/เรียง/แบ่งหน้า/empty มาตรฐานทุกหน้า list Must

As a ผู้ใช้ทุกแผนก, I want ทุกหน้า list มีตัวกรอง เรียง แบ่งหน้า และหน้าว่างที่คาดเดาได้เหมือนกัน, so that หาข้อมูลได้เร็วและใช้งานทุกหน้าแบบเดียวกันโดยไม่ต้องเรียนรู้ใหม่

Happy — filter + sort + pagination Given หน้า invoices มีใบแจ้งหนี้ 45 ใบ When กรองสถานะ = "เกินกำหนด" + เรียงตามวันครบกำหนด แล้วเปิดหน้า 2 Then แสดงเฉพาะใบเกินกำหนด เรียงใกล้สุดก่อน; pager "แสดง 21–40 จาก N"; page-size default 20 (เลือก 10/20/50 ได้)
Edge — คงตัวกรองเมื่อ refresh/กลับมา Given ผู้ใช้ตั้งตัวกรอง customers = สถานะ "ต้องติดตาม" + Sale "สมชาย" When refresh หน้า หรือกดเข้ารายละเอียดแล้วกดกลับ Then ตัวกรอง + หน้าปัจจุบันคงอยู่ (ไม่รีเซ็ตกลับหน้า 1/ไม่ล้างตัวกรอง)
Error/Empty — ไม่พบผล Given ตั้งตัวกรองที่ไม่มีข้อมูลตรง (เช่น qc ผล "ไม่ผ่าน" ที่ไม่มี) When ใช้ตัวกรอง Then แสดง empty state เฉพาะหน้า (เช่น "ไม่มีรายการรอตรวจ") + ปุ่ม "ล้างตัวกรอง" — ไม่ error, ไม่ตารางเปล่างงๆ
