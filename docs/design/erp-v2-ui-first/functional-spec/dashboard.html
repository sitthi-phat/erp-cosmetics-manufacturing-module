3. User Stories + Acceptance Criteria

US-DSH-01 — auto-refresh 15s คง view Must

As a ผู้ใช้แผนก, I want dashboard รีเฟรชอัตโนมัติทุก 15 วิ โดยค้าง view เดิม, so that เห็นข้อมูลล่าสุดโดยไม่เด้งกลับหน้าแรก

Happy Given อยู่ที่ dashboard ฝ่ายขาย + เปิด drill-down "ต้องติดตาม" When ครบ 15 วิ (auto) หรือกด refresh Then ตัวเลข/รายการอัปเดต แต่ยังอยู่ที่ view/drill-down เดิม (ไม่เด้งกลับ)
Edge — ปิด auto-refresh Given ผู้ใช้ปิด toggle auto-refresh When เวลาผ่าน 15 วิ Then ไม่รีเฟรชอัตโนมัติ (รีเฟรชเฉพาะกดปุ่ม)
Error — ไม่มีสิทธิ์แผนก Given ผู้ใช้ไม่มีสิทธิ์ Read ของแผนกนั้น When พยายามเปิด dashboard แผนกนั้น Then ไม่แสดงแผนกนั้นในตัวสลับ / 403

US-DSH-02 — KPI tile drill-down + context Must

As a ผู้ใช้, I want กด KPI tile แล้วเห็นรายการ + pagination และกดเข้า module พร้อม context, so that จากตัวเลขไปทำงานต่อได้เร็ว

Happy Given Stock dashboard tile "lot รอ QC = 3" When กด tile Then แสดงรายการ 3 lot + pagination; กดรายการ → เข้าหน้า QC/stock พร้อม lot ที่เลือก (context)
Edge — tile = 0 Given tile ค่าเป็น 0 When กด Then แสดง empty state "ไม่มีรายการ" (ไม่ error)
Error — date range ผิด Given เลือก date range เริ่ม > สิ้นสุด When ใช้ตัวกรอง Then error "ช่วงวันที่ไม่ถูกต้อง"
