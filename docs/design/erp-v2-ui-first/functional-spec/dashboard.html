Module: Dashboard รายแผนก (×7) Must

7 แผนก: Sale · Stock · Production · QC · Shipping · Finance · Admin · รวม 29 tile · auto-refresh 15s (default, คง view) · ทุก tile drill-down → list + pagination + deep link พร้อม context · การเห็นแผนก = ยึด permission Read ล้วนๆ (ไม่เกี่ยวชื่อ role)

ที่มาความจริง: นิยาม/สูตร/ชนิด (event/state) ต่อ tile ทั้ง 29 ตัว = po-spec-depth-audit §2.2 + status-journeys §12 · ป้ายชนิด: ในช่วง (event) = นับเหตุการณ์ที่เกิดในช่วง date-filter · ตอนนี้ (state) = snapshot ปัจจุบัน ไม่ขึ้นกับช่วง · ใกล้หมด = เทียบ Available (on_hand−Reserved) · มูลค่าสต็อก = จำนวน on_hand × ราคาซื้อล่าสุดของ Lot (ไม่หัก Reserved; COGS นอก scope — ปอนด์ยืนยัน) · Reservation = stock-reservation.md

1. ภาพรวม + Use cases

ผู้ใช้เห็น dashboard เฉพาะแผนกที่ตนมีสิทธิ์ Read (มี Read module ไหน = เห็น dashboard แผนกนั้น) · ผู้ที่มี Read หลาย module สลับแผนกได้ด้วย tab/selector · filter default = เดือนนี้ (เลือก วันนี้ / สัปดาห์นี้ / เดือนนี้ / กำหนดเอง) · แต่ละ tile ระบุ event vs state · KPI tile ทุกอัน drill-down เป็น list + pagination แล้วคลิกรายการเข้า module พร้อม context

Dataset (เดือนนี้): Sale ลูกค้าประจำ 2 · ห่างหาย 1 · ต้องติดตาม 1 · PO 11 · รอชำระ 1 | Stock ใกล้หมด (วิตามินอี) · lot รอ QC 3 · PR ค้าง 2 · มูลค่าสต็อกรวม | Production คิวงาน 11 (รอรับงาน 2 + ในสายผลิต 9) · rework 1 · Hold 1 | Shipping SHP-0044/45/46 | Finance รอวางบิล/ค้างชำระ/เกินกำหนด | Admin user/role/login/audit

3. User Stories + Acceptance Criteria

แกนพฤติกรรม (DSH-01..04) + 7 story รายแผนก (SALE/STK/PRD/QC/SHP/FIN/ADM) — แต่ละ story มีตารางนิยาม tile ครบทุกตัว + AC happy/edge/error

US-DSH-01 — auto-refresh 15s คง view/drill Must

As a ผู้ใช้แผนก, I want dashboard รีเฟรชอัตโนมัติทุก 15 วิ โดยค้าง view/drill เดิมและ filter เดิม, so that เห็นข้อมูลล่าสุดโดยไม่เด้งกลับหน้าแรกและไม่เสียตำแหน่งที่ดูอยู่

Happy Given อยู่ dashboard ฝ่ายขาย filter = "เดือนนี้" + เปิด drill-down tile "ต้องติดตาม" ค้างอยู่ When ครบ 15 วิ (auto) หรือกดปุ่ม refresh Then ตัวเลข/รายการอัปเดตค่าล่าสุด แต่ยังอยู่ที่ drill-down "ต้องติดตาม" + filter "เดือนนี้" เดิม (ไม่รีเซ็ตช่วง ไม่เด้งกลับ) และแสดง "อัปเดตล่าสุด HH:MM:SS"
Edge — ปิด auto-refresh Given ผู้ใช้ปิด toggle auto-refresh When เวลาผ่านไป 15 วิ / 30 วิ Then ตัวเลขไม่เปลี่ยนเอง (รีเฟรชเฉพาะเมื่อกดปุ่ม refresh); ป้าย "อัปเดตล่าสุด" คงเวลาเดิมจนกดเอง
Error — โหลด aggregate ล้มเหลว Given การเรียกข้อมูล dashboard ล้มเหลวชั่วคราว When ถึงรอบ refresh Then คงค่าล่าสุดที่แสดงอยู่ + แถบเตือน "อัปเดตไม่สำเร็จ ลองใหม่" (ไม่ล้างหน้าจอเป็นค่าว่าง/ไม่หน้าขาว)

US-DSH-02 — Tile drill-down → list + pagination + deep link พร้อม context Must

As a ผู้ใช้, I want กด KPI tile แล้วเห็นรายการเบื้องหลังตัวเลขพร้อมคอลัมน์ที่กำหนด และกดรายการเข้า module ต่อได้, so that จากตัวเลขไปทำงานต่อได้ทันทีโดยไม่ต้องไปค้นเอง

Happy Given Stock dashboard tile "ล็อตรอ QC = 3" When กด tile Then แสดง list 3 lot คอลัมน์ (lot / วัตถุดิบ) + pagination เมื่อเกินหนึ่งหน้า; คลิกแถว → เปิด qc.html โดยเลือก lot นั้นไว้ (context/deep link)
Edge — tile = 0 Given tile ค่าเป็น 0 (เช่น "รอทำใบคืน = 0") When กด tile Then แสดง empty state "ไม่มีรายการ" (ไม่ error, ไม่เปิด list ว่างเปล่าแบบสับสน)
Error — drill รายการที่ไม่มีสิทธิ์เข้า module ปลายทาง Given ผู้ใช้เห็น tile ผ่านสิทธิ์ Read แผนก แต่ไม่มีสิทธิ์ Read module ปลายทางของ deep link When คลิกแถวเพื่อ deep link Then guard 403 ที่หน้าปลายทาง (ไม่หลุดข้อมูลข้าม module)

US-DSH-03 — Date filter: presets + custom range + event/state Must

As a ผู้ใช้แผนก, I want เลือกช่วงเวลาของ dashboard (วันนี้/สัปดาห์นี้/เดือนนี้/กำหนดเอง) และเข้าใจว่า tile ไหนถูกกรองด้วยช่วง, so that อ่านตัวเลขได้ถูกความหมายและเจาะช่วงที่ต้องการได้

Happy Given dashboard filter default = "เดือนนี้" When เลือก chip "สัปดาห์นี้" Then ทุก tile ชนิด event ("ในช่วง") คำนวณใหม่ตามสัปดาห์นี้พร้อมกัน (เช่น "PO เดือนนี้" → PO ในสัปดาห์นี้); tile ชนิด state ("ตอนนี้") เช่น "ใกล้หมด", "ล็อตรอ QC" คงค่า snapshot ปัจจุบัน ไม่เปลี่ยนตามช่วง; caption ของแต่ละ tile ระบุ "ในช่วง"/"ตอนนี้" ชัดเจน
Edge — custom range + คง drill ที่เปิดอยู่ Given เลือก "กำหนดเอง" แล้วระบุช่วง 01/07–05/07 (หรือเลือกเดือน/ปี) ขณะเปิด drill "PO" ค้างอยู่ When ยืนยันช่วง Then tile event รีคำนวณตามช่วงใหม่ และ drill "PO" ที่เปิดอยู่รีคำนวณตามช่วงใหม่ด้วย (ไม่เด้งปิด drill)
Error — ช่วงกลับด้าน Given เลือก custom range เริ่ม > สิ้นสุด When กดใช้ตัวกรอง Then error "ช่วงวันที่ไม่ถูกต้อง" + ไม่ยิง query (คงค่าช่วงก่อนหน้า)

US-DSH-04 — เห็นแผนกตาม permission Read + สลับหลายแผนก Must

As a ผู้ใช้, I want เห็น dashboard เฉพาะแผนกที่ผมมีสิทธิ์ Read และสลับได้ถ้ามีหลายแผนก, so that เห็นเฉพาะข้อมูลที่มีสิทธิ์ ไม่เกี่ยวว่าถูกตั้งชื่อ role อะไร

Happy — ยึด permission ล้วน (ปอนด์ยืนยัน) Given ผู้ใช้มีสิทธิ์ Read เฉพาะ module Sale และ Finance (ไม่ว่าชื่อ role คืออะไร) When เปิด dashboard Then ตัวสลับแผนกแสดง เฉพาะ Sale + Finance (2 แท็บ); แผนกที่ไม่มี Read ไม่ปรากฏในตัวสลับ; เริ่มที่แผนกแรกที่มีสิทธิ์
Edge — มี Read module เดียว / ครบทุก module Given ผู้ใช้มี Read แผนกเดียว (เช่น Production เท่านั้น) — หรืออีกกรณีมี Read ครบทุก module When เปิด dashboard Then กรณีแผนกเดียว: แสดงแผนกนั้นโดยไม่มีตัวสลับ; กรณีมีครบ: เห็นทั้ง 7 แผนกในตัวสลับ (superset ตามสิทธิ์ ไม่ใช่ตามชื่อ Admin)
Error — เข้าถึง URL แผนกที่ไม่มีสิทธิ์ Given ผู้ใช้ไม่มีสิทธิ์ Read module ของแผนกหนึ่ง When พิมพ์ URL/parameter ของ dashboard แผนกนั้นตรงๆ Then guard ปฏิเสธ 403 (ไม่แสดงตัวเลข/รายการของแผนกที่ไม่มีสิทธิ์)

US-DSH-SALE — Dashboard ฝ่ายขาย (5 tile) · Read Sale Must

As a ผู้มีสิทธิ์ Read Sale, I want เห็นสถานะลูกค้า/ออเดอร์/การชำระของฝ่ายขายเป็น KPI พร้อม drill, so that ตามงานขายและลูกค้าที่ต้องดูแลได้ครบ

tileสูตรนับ / เงื่อนไขเข้ากลุ่มชนิดdrill (คอลัมน์ → ปลายทาง)
ลูกค้าประจำcount(customer.status=ลูกค้าประจำ(Active) AND มี order ในช่วง)eventชื่อลูกค้า/สถานะ → customer-detail
ห่างหายcount(customer เปลี่ยน→ห่างหาย(Inactive) ในช่วง โดย scheduler; เกณฑ์ = inactive_after_months ของลูกค้า default 3, ชุด {1,3,6,8})eventชื่อ/เหตุ (ไม่มี order เกิน N เดือน) → customer-detail
POcount(po.created_date อยู่ในช่วง)eventPO/ลูกค้า/สถานะ → po-detail
รอชำระcount(invoice.status=รอชำระ AND ยังไม่เกินกำหนด AND issued ในช่วง)eventINV/ลูกค้า/ครบกำหนด → invoice-detail
ต้องติดตามcount(customer.status=ต้องติดตาม(Follow-up) ที่ยังค้าง)stateชื่อ/เหตุผล(comment) → customer-detail
Happy — สูตร + drill Given เดือนนี้มีลูกค้าประจำที่มี order 2 ราย และ PO 11 ใบ When ดู tile "ลูกค้าประจำ" แล้วกด Then tile = 2 (event, caption "ในช่วง"); drill แสดง 2 แถว (ชื่อลูกค้า/สถานะ) → คลิกไป customer-detail ของรายนั้น
Edge — "ต้องติดตาม" เป็น state ไม่ขึ้นกับช่วง Given tile "ต้องติดตาม" = 1 (เดอร์มา แคร์ จาก PO-188 Hold) และผู้ใช้เปลี่ยน filter เป็น "วันนี้" When เปลี่ยนช่วง Then tile "ต้องติดตาม" ยังคง = 1 (state, caption "ตอนนี้" ไม่เปลี่ยนตามช่วง); drill แสดงเหตุผลจาก comment
Error — ไม่มี Read Sale Given ผู้ใช้ไม่มีสิทธิ์ Read Sale When เปิด dashboard Then ไม่เห็นแผนก Sale ในตัวสลับ / เข้า URL ตรง = 403

US-DSH-STK — Dashboard ฝ่ายคลัง (4 tile: ใกล้หมด=Available, มูลค่า=on_hand) · Read Stock Must

As a ผู้มีสิทธิ์ Read Stock, I want เห็นวัตถุดิบใกล้หมด (ตามยอดใช้ได้)/ล็อตรอ QC/PR ค้าง และมูลค่าสต็อกรวม, so that เติมของทัน ตรวจรับทัน และรู้เงินจมในคลัง

tileสูตร/เงื่อนไขชนิดdrill
ใกล้หมดcount(วัตถุดิบที่ Available (on_hand − Reserved) ≤ low_stock_threshold; เฉพาะที่ตั้ง threshold — null = ไม่นับ)stateวัตถุดิบ/ใช้ได้ (+คงคลัง/จองแล้ว) → stock
ล็อตรอ QCcount(lot.qc_status=รอตรวจรับ)statelot/วัตถุดิบ → qc
คำขอ PR ค้างcount(PR.status ∈ {เปิดคำขอ, รับทราบ, รับบางส่วน})statePR/วัตถุดิบ/สถานะ → purchase-request
มูลค่าสต็อกΣ(on_hand ต่อ lot × ราคาซื้อล่าสุดของ lot นั้น) — เป็นยอดเงิน · ไม่หัก Reserved (ของยังอยู่ในคลัง; COGS นอก scope); lot on_hand ติดลบหักลบยอดstateวัตถุดิบมูลค่าสูงสุด → stock
Happy — สูตรมูลค่าสต็อก (on_hand ไม่หักจอง) Given วัตถุดิบ A มี 2 lot: lot เก่า 10 กก. ราคาซื้อ 40, lot ใหม่ 5 กก. ราคาซื้อ 45 · ถูกจอง 8 กก. When ดู tile "มูลค่าสต็อก" Then มูลค่าของ A คิดต่อ lot จาก on_hand = (10×40)+(5×45) = 625 (ไม่หัก Reserved 8; แต่ละ lot ใช้ราคาซื้อล่าสุดของ lot ตัวเอง); tile รวมทุกวัตถุดิบทั้งคลัง; drill เรียงจากมูลค่าสูงสุด → stock
Edge — "ใกล้หมด" ใช้ Available + เฉพาะที่ตั้ง threshold Given วัตถุดิบ X ไม่ได้ตั้ง threshold (null) แม้ Available ต่ำ; วัตถุดิบ Y on_hand=66 แต่ถูกจอง 70 → Available −4; วัตถุดิบ Z on_hand เพียงพอแต่ถูกจองจน Available ≤ threshold When ดู tile "ใกล้หมด" (state) Then X ไม่ถูกนับ (null=ไม่เตือน); Y และ Z ถูกนับ เพราะ Available ≤ threshold (นับตามยอดที่รับปากได้จริง ไม่ใช่ on_hand); มูลค่าสต็อกยังคิดจาก on_hand ปกติ
Error — ไม่มี Read Stock Given ผู้ใช้ไม่มีสิทธิ์ Read Stock When เปิด dashboard Then ไม่เห็นแผนก Stock / URL ตรง = 403
