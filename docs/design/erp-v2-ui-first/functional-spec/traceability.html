3. User Stories + Acceptance Criteria

US-TRC-01 — Entity selector: ค้นได้อะไร + field ที่ค้นได้ต่อ entity Must

As a ผู้ตรวจสอบ, I want เลือก entity แล้วรู้ว่าค้นด้วย field ใดได้บ้าง, so that ค้นเจอเป้าหมายได้ตรงโดยไม่ต้องเดา

Entity ที่ค้นได้field ที่ค้นได้
ลูกค้า (Customer)รหัส CUS / ชื่อ / เบอร์ / สถานะ
POเลข PO / ลูกค้า / สถานะ fulfilment/billing
PRD / Batchเลข PRD / เลข Batch (B-{PO}-{line}-{run}) / สถานะ
วัตถุดิบ / Lotรหัส RM / ชื่อ / เลข Lot / supplier / qc_status
PR / GRเลข PR / เลข GR / วัตถุดิบ / สถานะ
Supplierรหัส SUP / ชื่อ / สถานะ active
Shipment / DN / Invoiceเลข SHP / DN / INV / ลูกค้า / สถานะ
Happy Given เลือก entity = Batch When ค้นด้วยเลข Batch "B-PO-202607-000170-1-2" Then พบ Batch นั้น + ปุ่มดู genealogy; ช่องค้นแสดง field ที่ใช้ได้ของ Batch (เลข PRD/เลข Batch/สถานะ)
Edge — date range + time Given กรองด้วยช่วงวัน + เวลา ร่วมกับ entity When ค้น Then แสดงเฉพาะรายการในช่วง; ถ้าไม่พบ = empty state "ไม่พบข้อมูลตามเงื่อนไข"
Error — ไม่มีสิทธิ์ Read entity Given ผู้ใช้ไม่มีสิทธิ์ Read module ของ entity นั้น When ค้น Then entity นั้นไม่อยู่ใน selector / ผลไม่แสดง (403 ตามสิทธิ์ Read)

US-TRC-02 — Genealogy (Lot→Batch→FG) + คลิก node ไปหน้าจริง Must

As a ผู้ตรวจสอบ GMP, I want เห็นสายพันธุ์การผลิตและคลิก node ไปหน้าที่เกี่ยวข้อง, so that สืบย้อนกลับได้ครบและเปิดดูรายละเอียดต้นทางได้

Happy — genealogy + node คลิกได้ Given ค้น B-PO-202607-000170-1-2 When เปิด genealogy Then แสดงสาย: Lot (L-GLY-2506, L-OLV-2604) → Batch run1(ไม่ผ่าน)+run2(ผ่าน) → line1 → PO-170 → กลอรี่ → DN/INV; คลิก node Lot → stock/qc ของ lot, node Batch → production/qc, node PO → po-detail, node DN/INV → หน้าเอกสารนั้น
Edge — อธิบาย Lot vs Batch Given ผู้ใช้ใหม่ดู genealogy When hover/ดูคำอธิบาย Then แสดงคำอธิบาย "Lot = ล็อตวัตถุดิบรับเข้า · Batch = รอบผลิต 1 line (rework = Batch run ใหม่)" (ดู Glossary)
Error — entity ไม่มีสายผลิต Given ค้น entity ที่ไม่มี genealogy (เช่น Supplier) When เปิด genealogy Then แสดง "รายการนี้ไม่มีสายการผลิต" + ยังดู field-audit ได้ (ไม่ error หน้าค้าง)

US-TRC-03 — ตาราง Field-level audit (คอลัมน์/filter/sort/pagination) + archive Must

As a ผู้ตรวจสอบ, I want เห็นการแก้ทุกค่าระดับ field ในตารางที่กรอง/เรียง/แบ่งหน้าได้ และ Super User archive ได้, so that ตรวจสอบใครทำอะไรและเก็บระยะยาว

Happy — คอลัมน์ + filter + sort Given CUS-000021 ถูกตั้ง Follow-up และมี audit หลายรายการ When เปิดตาราง field-audit แล้วกรอง field = "status" + เรียงเวลาใหม่→เก่า Then แสดงคอลัมน์ (เวลา / ผู้ทำ / entity / field / จาก→เป็น / เหตุผล) — แถว: สมหญิง / CUS-000021 / status / Active→Follow-up / เหตุผล; มี pagination เมื่อเกินหนึ่งหน้า; sort เวลา default ใหม่→เก่า
Edge — archive (Super User) Given ผู้ใช้เป็น Super User เลือกช่วงวันที่ When กด "archive เป็น text file" Then export ช่วงที่เลือกเป็น text file (retention online 1 ปี; หลังจากนั้น purge/archive manual — ไม่มี auto-purge) + ยืนยันก่อน export
Error — archive โดยไม่ใช่ Super User Given ผู้ใช้ทั่วไป (ไม่ใช่ Super User) When พยายาม archive Then ไม่มีปุ่ม archive / 403 (archive = Super User เท่านั้น)
