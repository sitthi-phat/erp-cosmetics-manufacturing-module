3. User Stories + Acceptance Criteria

US-TRC-01 — Search + genealogy (Lot→Batch→FG) Must

As a ผู้ตรวจสอบ, I want ค้นทุก entity และไล่ genealogy การผลิต, so that สืบย้อนกลับ GMP ได้ครบ

Happy Given เลือก entity = Batch, ค้น B-PO-202607-000170-1-2 When ค้นหา Then แสดง genealogy: Lot(L-GLY-2506, L-OLV-2604) → Batch run1(ไม่ผ่าน)+run2(ผ่าน) → line1 → PO-170 → กลอรี่ → DN/INV
Edge — date range + time Given กรองด้วยช่วงวัน+เวลา When ค้น Then แสดงเฉพาะรายการในช่วง; ถ้าไม่พบ = empty state
Error — ไม่มีสิทธิ์ Read Given ผู้ใช้ไม่มีสิทธิ์ Read entity นั้น When ค้น Then ไม่แสดงผล entity ที่ไม่มีสิทธิ์ / 403

US-TRC-02 — Field-level audit + archive (Super User) Must

As a ผู้ตรวจสอบ, I want เห็นการแก้ทุกค่าระดับ field และ Super User archive ได้, so that ตรวจสอบใครทำอะไรและเก็บระยะยาว

Happy Given CUS-000021 ถูกตั้ง Follow-up When เปิด field-audit Then แสดงแถว: เวลา/ผู้ทำ(สมหญิง)/entity(CUS-000021)/field(status)/จาก(Active)→เป็น(Follow-up)/เหตุผล
Edge — archive Given ผู้ใช้เป็น Super User When กด "archive เป็น text file" Then ระบบ archive ช่วงที่เลือก (retention online 1 ปี, purge/archive manual โดย Super User)
Error — archive โดยไม่ใช่ Super User Given ผู้ใช้ทั่วไป When พยายาม archive Then ไม่มีปุ่ม / 403 (archive = Super User เท่านั้น)
