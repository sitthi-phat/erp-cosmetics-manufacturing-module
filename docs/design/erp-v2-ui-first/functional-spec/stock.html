3. User Stories + Acceptance Criteria

US-STK-01 — เพิ่มวัตถุดิบใหม่ + UOM + เกณฑ์ใกล้หมด Must

As a ฝ่ายคลัง, I want เพิ่มวัตถุดิบที่ไม่เคยมี พร้อมหน่วยนับและเกณฑ์ใกล้หมด, so that รับเข้าของใหม่ได้ถูกต้องและเตือนสต็อกตามที่ตั้ง

Happy Given ฝ่ายคลังอยู่หน้า stock When เพิ่ม "วิตามินอี" UOM=กก. เกณฑ์ใกล้หมด = 2 กก. Then ระบบสร้างวัตถุดิบ RM-xxx; เมื่อยอดคงเหลือ ≤ 2 กก. จะแสดง "ใกล้หมด" ใน Stock Dashboard
Edge — ไม่ตั้งเกณฑ์ (ไม่มี default) Given เพิ่มวัตถุดิบโดยไม่กรอกเกณฑ์ใกล้หมด When ยอดคงเหลือลดลงมาก Then ระบบไม่เตือน "ใกล้หมด" (ไม่มี default) จนกว่าจะตั้งค่า
Error — ไม่ระบุ UOM / ชื่อซ้ำ Given เว้น UOM ว่าง หรือใช้ชื่อวัตถุดิบซ้ำที่มีอยู่ When บันทึก Then error "ต้องระบุหน่วยนับ" / "วัตถุดิบนี้มีอยู่แล้ว" — ไม่สร้างซ้ำ

US-STK-02 — Goods Receipt multi-line + ปิด PR อัตโนมัติ Must

As a ฝ่ายคลัง, I want รับเข้าหลายรายการในใบเดียว อ้างเลข PR ได้, so that เข้าคลังพร้อม gen Lot และปิด PR ที่รับครบอัตโนมัติ

Happy Given ฝ่ายคลังทำ GR ของ SUP-01 อ้าง PR-000031 (กลีเซอรีน 6 กก.) When บันทึก GR-20260708-007 Then gen Lot L-GLY-2607 (สถานะ "รอตรวจรับ"); PR-000031 → "ของเข้าครบ (Fulfilled)" อัตโนมัติ พร้อมเหตุผลว่ารับจาก lot ไหน — ดู Cascade C17
Edge — รับบางส่วน → เสนอ PR ใหม่ Given GR รับแอลกอฮอล์ 200/250 ล. อ้าง PR-000028 When บันทึก GR-20260708-008 Then PR-000028 = "รับบางส่วน"; ระบบเปิดกล่องยืนยัน "สร้าง PR ใหม่ 50 ล.? (PR-000032)" รอ user review (ยังไม่สร้างจนกด) — ดู PR
Error — ไม่กรอกเลขใบรับ/supplier Given เว้น supplier หรือเลขใบรับจาก supplier ว่าง When บันทึก GR Then error ที่ field ที่ขาด — ไม่บันทึก, ไม่ gen Lot

US-STK-03 — Negative stock + บวกกลับตอนรับเข้า Must

As a ฝ่ายคลัง, I want เห็นยอดวัตถุดิบที่ติดลบจากการผลิตล่วงหน้า และให้ GR บวกกลับชดเชย, so that รู้ว่ามีของค้างรับและยอดถูกต้องเมื่อของเข้า

Happy Given การผลิตตัดกลีเซอรีนจนยอดคงเหลือ = −4 กก. When เปิดหน้า stock Then แสดงยอด −4 เป็นตัวเลขสีแดง + badge "ติดลบ (รอรับเข้า)" + ปริมาณติดลบ
Edge — GR ชดเชยยอดติดลบ Given ยอดกลีเซอรีน −4 กก. และรับเข้า 6 กก. When ยืนยัน GR Then ก่อนยืนยันแสดง notice "การรับนี้ชดเชยยอดติดลบ 4 หน่วย"; หลังรับ ยอดคงเหลือ = +2 (ชดเชย 4 ก่อน แล้วส่วนเกิน 2 เข้าปกติ) + trace
Error — ราคาซื้อติดลบ Given กรอกราคาซื้อเป็นค่าติดลบ When บันทึก Then error "ราคาต้อง ≥ 0" (0 ได้ แต่ติดลบไม่ได้) — ไม่บันทึก

US-STK-04 — ราคาซื้อ/ขาย 0 ได้ + FIFO Should

As a ฝ่ายคลัง, I want กำหนดราคาซื้อและราคาขายแยกกัน (0 ได้) และให้ระบบเลือก Lot แบบ FIFO, so that ราคาขาย default ไป PO และการตัด stock ถูกต้องตามลำดับล็อต

Happy Given รับเข้าวัตถุดิบพร้อมราคาซื้อ 45 ราคาขาย 0 When บันทึก Then ระบบยอมรับราคาขาย 0; ราคาขายใช้เป็น default ตอนขายวัตถุดิบใน PO
Edge — FIFO หลาย Lot Given วัตถุดิบมี Lot เก่า L-GLY-2506 และใหม่ L-GLY-2607 When การผลิตตัด stock Then ระบบตัดจาก Lot เก่าก่อน (FIFO) และ trace ระบุ Lot ที่ใช้
Error — แก้ threshold เป็นค่าลบ Given ตั้งเกณฑ์ใกล้หมดเป็นค่าติดลบ หรือ % > 100 When บันทึก Then error "เกณฑ์ต้องเป็นค่าที่ถูกต้อง (จำนวน ≥ 0 หรือ % 0–100)"
