Module: คลังวัตถุดิบ + Goods Receipt Tier 1

เพิ่มวัตถุดิบ+UOM+เกณฑ์ใกล้หมด (ไม่มี default) · ราคาซื้อ/ขาย 0 ได้ · Goods Receipt multi-line → gen Lot + ปิด PR + บวก stock กลับ/ชดเชยยอดติดลบ · 3 ยอด: คงคลัง/จองแล้ว/ใช้ได้ · มูลค่าสต็อก = on_hand × ราคาซื้อล่าสุดของ Lot · เลข Lot {supplier prefix}{YYMM}, GR GR-{YYYYMMDD}-{NNN}

3 ยอดต่อวัตถุดิบ (ปอนด์ล็อก 2026-07-10): คงคลัง (on_hand) = ยอดกายภาพจริง (Σ movement ราย lot; ติดลบได้จากผลิตล่วงหน้า) · จองแล้ว (Reserved) = Σ reservation ที่ยัง active (≥0) · ใช้ได้ (Available) = on_hand − Reserved (ติดลบได้จากจองเกิน) · **negative 2 ชนิดต้องแยก:** Available<0 = "จองเกิน (รอรับเข้า)" · on_hand<0 = "ติดลบ (รอรับเข้า)" — ทั้งคู่เตือนไม่บล็อก · ที่มา: stock-reservation.md

1. ภาพรวม + Use cases

ฝ่ายคลังจัดการวัตถุดิบ+Lot, รับเข้า (Goods Receipt), เห็น 3 ยอด (คงคลัง/จองแล้ว/ใช้ได้) รวมยอดติดลบ 2 ชนิด และดูมูลค่าสต็อก · เกณฑ์ "ใกล้หมด" ตั้งต่อวัตถุดิบ (% หรือจำนวน) เทียบ Available — ไม่มี default ไม่เตือนจนกว่าตั้งค่า · ไม่มีฟังก์ชันรับเข้าในหน้า Supplier (อยู่ที่นี่)

3. User Stories + Acceptance Criteria

US-STK-01 — เพิ่มวัตถุดิบใหม่ + UOM + เกณฑ์ใกล้หมด (เทียบ Available) Must

As a ฝ่ายคลัง, I want เพิ่มวัตถุดิบที่ไม่เคยมี พร้อมหน่วยนับและเกณฑ์ใกล้หมด, so that รับเข้าของใหม่ได้ถูกต้องและเตือนสต็อกตามยอดที่ใช้ได้จริง (หักจองแล้ว)

Happy Given ฝ่ายคลังอยู่หน้า stock When เพิ่ม "วิตามินอี" UOM=กก. เกณฑ์ใกล้หมด = 2 กก. Then ระบบสร้างวัตถุดิบ RM-xxx; เมื่อ Available (on_hand − Reserved) ≤ 2 กก. จะแสดง "ใกล้หมด" ใน Stock Dashboard (ดู US-DSH-STK)
Edge — ไม่ตั้งเกณฑ์ (ไม่มี default) Given เพิ่มวัตถุดิบโดยไม่กรอกเกณฑ์ใกล้หมด When Available ลดลงมาก (แม้จองจนติดลบ) Then ระบบไม่เตือน "ใกล้หมด" (ไม่มี default) จนกว่าจะตั้งค่า
Error — ไม่ระบุ UOM / ชื่อซ้ำ Given เว้น UOM ว่าง หรือใช้ชื่อวัตถุดิบซ้ำที่มีอยู่ When บันทึก Then error "ต้องระบุหน่วยนับ" / "วัตถุดิบนี้มีอยู่แล้ว" — ไม่สร้างซ้ำ

US-STK-02 — Goods Receipt multi-line + ปิด PR อัตโนมัติ Must

As a ฝ่ายคลัง, I want รับเข้าหลายรายการในใบเดียว อ้างเลข PR ได้, so that เข้าคลังพร้อม gen Lot และปิด PR ที่รับครบอัตโนมัติ

Happy Given ฝ่ายคลังทำ GR ของ SUP-01 อ้าง PR-000031 (กลีเซอรีน 6 กก.) When บันทึก GR-20260708-007 Then gen Lot L-GLY-2607 (สถานะ "รอตรวจรับ"); on_hand เพิ่ม → Available เพิ่มอัตโนมัติ (ไม่แตะ Reserved); PR-000031 → "ของเข้าครบ (Fulfilled)" อัตโนมัติ — ดู Cascade C17
Edge — รับบางส่วน → เสนอ PR ใหม่ Given GR รับแอลกอฮอล์ 200/250 ล. อ้าง PR-000028 When บันทึก GR-20260708-008 Then PR-000028 = "รับบางส่วน"; ระบบเปิดกล่องยืนยัน "สร้าง PR ใหม่ 50 ล.? (PR-000032)" รอ user review (ยังไม่สร้างจนกด) — ดู PR
Error — ไม่กรอกเลขใบรับ/supplier Given เว้น supplier หรือเลขใบรับจาก supplier ว่าง When บันทึก GR Then error ที่ field ที่ขาด — ไม่บันทึก, ไม่ gen Lot

US-STK-03 — 3 ยอด + ยอดติดลบ 2 ชนิด (จองเกิน / กายภาพ) + GR บวกกลับ Must

As a ฝ่ายคลัง, I want เห็น 3 ยอด (คงคลัง/จองแล้ว/ใช้ได้) และแยกให้ชัดว่าติดลบเพราะจองเกินหรือเพราะผลิตล่วงหน้า และให้ GR บวกกลับ, so that เข้าใจสถานะจริงและรู้ว่ามีของค้างรับ

Happy — 3 ยอด + จองเกิน (Available ติดลบ) Given กลีเซอรีน on_hand = 66, ถูกจองโดย PO = 70 When เปิดหน้า stock Then แสดง คงคลัง 66 · จองแล้ว 70 · ใช้ได้ −4; ช่อง "ใช้ได้" เป็น แดง + badge "จองเกิน (รอรับเข้า)" (on_hand ยัง 66 ปกติ ไม่แดง)
Edge — on_hand ติดลบ (ผลิตล่วงหน้า) + GR ชดเชย + FIFO retro-link Given การผลิตตัด on_hand กลีเซอรีนจนเหลือ −4 กก. แล้วรับเข้า 6 กก. When เปิด stock / ยืนยัน GR Then on_hand −4 = แดง + badge "ติดลบ (รอรับเข้า)"; ก่อนยืนยัน GR แสดง notice "การรับนี้ชดเชยยอดติดลบ 4 หน่วย"; หลังรับ on_hand = +2; ระบบ back-allocate การตัดที่ติดลบเข้า lot ใหม่แบบ FIFO (คง Batch↔Lot GMP) + trace
Error — ราคาซื้อติดลบ Given กรอกราคาซื้อเป็นค่าติดลบ When บันทึก Then error "ราคาต้อง ≥ 0" (0 ได้ แต่ติดลบไม่ได้) — ไม่บันทึก

US-STK-04 — ราคาซื้อ/ขาย 0 ได้ + FIFO Should

As a ฝ่ายคลัง, I want กำหนดราคาซื้อและราคาขายแยกกัน (0 ได้) และให้ระบบเลือก Lot แบบ FIFO, so that ราคาขาย default ไป PO และการตัด stock ถูกต้องตามลำดับล็อต

Happy Given รับเข้าวัตถุดิบพร้อมราคาซื้อ 45 ราคาขาย 0 When บันทึก Then ระบบยอมรับราคาขาย 0; ราคาขายใช้เป็น default ตอนขายวัตถุดิบใน PO
Edge — FIFO หลาย Lot Given วัตถุดิบมี Lot เก่า L-GLY-2506 และใหม่ L-GLY-2607 When การผลิตตัด stock (consume) Then ระบบตัดจาก Lot เก่าก่อน (FIFO) และ trace ระบุ Lot ที่ใช้
Error — แก้ threshold เป็นค่าลบ Given ตั้งเกณฑ์ใกล้หมดเป็นค่าติดลบ หรือ % > 100 When บันทึก Then error "เกณฑ์ต้องเป็นค่าที่ถูกต้อง (จำนวน ≥ 0 หรือ % 0–100)"

US-STK-05 — มูลค่าสต็อก (on_hand × ราคาซื้อล่าสุดของ Lot · ไม่หัก Reserved) Should

As a ฝ่ายคลัง/ผู้บริหาร, I want เห็นมูลค่าสต็อกต่อวัตถุดิบและรวมทั้งคลัง, so that รู้เงินจมในคลังโดยไม่ต้องทำ COGS เต็มรูป

สูตร (ปอนด์ล็อก): มูลค่าคิดจาก คงคลัง (on_hand กายภาพ) เท่านั้น — ไม่หัก Reserved (ของยังอยู่ในคลังจริง แค่ถูกจองไว้) · มูลค่าต่อ Lot = on_hand ของ lot × ราคาซื้อล่าสุดของ lot นั้น · รวมวัตถุดิบ = Σ ทุก lot · รวมคลัง = Σ ทุกวัตถุดิบ · COGS/valuation FIFO เต็มรูป = นอก scope
Happy — สูตรต่อ lot (ไม่หักจอง) Given กลีเซอรีนมี lot เก่า 10 กก. ราคาซื้อ 40 และ lot ใหม่ 5 กก. ราคาซื้อ 45 · ถูกจองไป 8 กก. When เปิดหน้า stock ดูมูลค่า Then มูลค่ากลีเซอรีน = (10×40)+(5×45) = 625 บาท (คิดจาก on_hand 15 ไม่หัก Reserved 8); รวมคลัง = Σ ทุกวัตถุดิบ (ตรงกับ tile "มูลค่าสต็อก" ใน Dashboard)
Edge — lot on_hand ติดลบ / ราคาซื้อ 0 Given แอลกอฮอล์มี lot on_hand ติดลบ −30 และวัตถุดิบบางตัวราคาซื้อ = 0 When คิดมูลค่า Then lot on_hand ติดลบหัก(ลด)มูลค่ารวม (−30 × ราคาซื้อล่าสุด); วัตถุดิบราคาซื้อ 0 = มูลค่า 0 · Reserved ไม่กระทบมูลค่า (ไม่ error)
Error — ไม่มีสิทธิ์ Read Stock Given ผู้ใช้ไม่มีสิทธิ์ Read Stock When พยายามเปิดหน้า stock / tile มูลค่าสต็อก Then guard 403 (มูลค่าสต็อกเป็นข้อมูลไวต่อธุรกิจ — เห็นตามสิทธิ์ Read Stock เท่านั้น)

US-STK-06 — มุมมอง "วัตถุดิบถูกจองโดย PO ใด" (Reservation view) Could

As a ฝ่ายคลัง/จัดซื้อ, I want ดูว่าวัตถุดิบที่ถูกจองอยู่ ถูกจองโดย PO/line ใดบ้าง, so that เข้าใจว่าทำไม Available ต่ำ/ติดลบ และวางแผนสั่งซื้อได้

Happy — drill "จองแล้ว" Given กลีเซอรีน จองแล้ว 70 (จาก PO-181 line1 = 2, PO-188 = 2.4, PO-185 = ...) When คลิกยอด "จองแล้ว" ของกลีเซอรีน Then แสดงรายการ reservation ที่ยัง active: PO/line/จำนวนจอง/สถานะ (Reserved) + deep link → po-detail; รวมยอดตรงกับ "จองแล้ว 70"
Edge — ไม่มีการจอง Given วัตถุดิบที่ Reserved = 0 When คลิก/ดูมุมมองจอง Then empty state "ยังไม่มี PO จองวัตถุดิบนี้" (Available = on_hand); ไม่ error
Error — reservation ที่ consume/release ไปแล้ว Given reservation ที่ถูก consume (เริ่มผลิตแล้ว) หรือ released (PO cancel) When ดูมุมมองจอง (active เท่านั้น) Then รายการที่ consume/release ไม่นับใน "จองแล้ว" (แสดงเฉพาะ active) — ดูประวัติได้ที่ trace
