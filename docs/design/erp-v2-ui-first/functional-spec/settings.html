3. User Stories + Acceptance Criteria

US-SET-01 — Role & สิทธิ์ (RUCDAA) + สร้าง role Must

As a Admin, I want สร้าง role และตั้งสิทธิ์ RUCDAA ราย module, so that ควบคุมการเข้าถึงได้ยืดหยุ่น

Happy Given Admin อยู่แท็บ Role When สร้าง role "หัวหน้าคลัง" ตั้ง Stock=R,U,C / Production=R Then บันทึก role + matrix 6 ระดับ (R/U/C/D/Approve/Admin) ราย module
Edge — Read = เห็น + รับ noti Given role มี Read ของ module Production When มีงานส่งเข้า Production Then ผู้ใช้ role นี้เห็น module + ได้รับ notification ของ module นั้น (ดู Notification)
Error — ลบ role ที่มี user Given role มี user ผูกอยู่ When พยายามลบ role Then บล็อก + เสนอ "ย้าย user ไป role อื่นก่อน" (ดู Deletion §2.7)

US-SET-02 — Config VAT + effective date Must

As a Admin, I want ตั้ง VAT พร้อมวันที่มีผล, so that ใบกำกับภาษีคิดอัตราถูกตามช่วงเวลา

Happy Given แท็บ Config VAT When ตั้ง VAT 7% effective 01/01/2569 Then บันทึก + เก็บประวัติ; ใบกำกับภาษียึดอัตราตาม invoice date (ดู US-INV-02)
Edge — ข้อมูลบริษัท (ใบกำกับ) Given แท็บข้อมูลบริษัท When กรอกชื่อ/เลขผู้เสียภาษี/ที่อยู่/logo Then ข้อมูลนี้ปรากฏบน invoice-print
Error — effective date ทับซ้อน/ว่าง Given effective date ว่างหรือทับช่วงเดิม When บันทึก Then error "ต้องระบุวันที่มีผลที่ไม่ทับซ้อน"
