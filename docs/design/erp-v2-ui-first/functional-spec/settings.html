Module: Settings (5 หน้าจอ) Must

5 แท็บ: 1) Role & สิทธิ์ (RUCDAA + Admin bit) 2) ผู้ใช้ (สร้าง/ผูก Google/เปิด-ปิด/เปลี่ยน role/ลบ→bulk reassign) 3) Config VAT + effective date 4) ข้อมูลบริษัท 5) Audit log · 1 story ต่อหน้าจอ

ที่มา: deletion กติกา (ปอนด์ยืนยัน) = rbac-deletion.html · ลบ Sale ต้อง bulk-reassign ลูกค้าก่อน; ลบ Role บล็อกจนย้าย user ออกหมด · RUCDAA 6 ระดับต่อ module = Read/Update/Create/Delete/Approve/Admin · VAT effective ยึด invoice date

1. ภาพรวม + Use cases

ผู้มีสิทธิ์ Admin/Read Settings ตั้งค่าสิทธิ์ราย module (RUCDAA), จัดการ user ใต้ role (สร้าง/ผูก Google/เปิด-ปิด/ย้าย role/ลบพร้อมย้ายงาน), ตั้ง VAT + effective date, กรอกข้อมูลบริษัท (ใช้บนใบกำกับภาษี), และดู audit log ทั้งระบบ

3. User Stories + Acceptance Criteria

US-SET-01 — หน้าจอ 1: Role & สิทธิ์ (RUCDAA + Admin bit) Must

As a ผู้ดูแลระบบ (Admin), I want สร้าง role และตั้งสิทธิ์ RUCDAA 6 ระดับต่อ module รวม Admin bit, so that ควบคุมการเข้าถึงได้ยืดหยุ่นโดยไม่ต้องแก้โค้ด

Happy Given Admin อยู่แท็บ Role กด "สร้าง role" When ตั้ง role "หัวหน้าคลัง" — Stock = R,U,C ; Production = R ; ที่เหลือว่าง แล้วบันทึก Then บันทึก role พร้อม matrix 6 ช่อง (R/U/C/D/Approve/Admin) ต่อทุก module; ผู้ใช้ role นี้เห็น Stock+Production, สร้าง/แก้ Stock ได้, แต่ลบ/approve ไม่ได้
Edge — Admin bit = force override Given role ที่ติ๊ก Admin bit ของ module PO When ผู้ใช้ role นั้นเปิด po-detail Then เห็นตัวเลือก "เปลี่ยนสถานะข้ามลำดับ (force override)" (ดู US-PO-06); role ที่ไม่มี Admin bit ไม่เห็นตัวเลือกนี้
Error — ลบ role ที่มี user Given role มี user ผูกอยู่ ≥ 1 คน When กดลบ role Then บล็อก + ข้อความ "ต้องย้าย user ออกจาก role นี้ให้หมดก่อนลบ" (no force-migrate) — ดู Deletion Role

US-SET-02 — หน้าจอ 2: จัดการผู้ใช้ (สร้าง/Google/เปิด-ปิด/เปลี่ยน role/ลบ→bulk reassign) Must

As a Admin, I want สร้าง user ผูกบัญชี Google เปิด-ปิดการใช้งาน เปลี่ยน role และลบ user โดยบังคับย้ายงาน/ลูกค้าก่อน, so that จัดการบัญชีได้ครบโดยไม่มีลูกค้า/งานตกค้างไม่มีเจ้าภาพ

Happy — สร้าง + ผูก Google + เปลี่ยน role Given Admin อยู่แท็บ "ผู้ใช้" กด "เพิ่มผู้ใช้" When กรอกชื่อ/อีเมล เลือก role "พนักงานขาย" ผูกบัญชี Google และตั้ง Active Then สร้าง user ล็อกอินได้ทั้ง local + Google; แถวผู้ใช้แสดง role + สถานะ Active; เปลี่ยน role/สลับ Active↔ปิดใช้งานได้จากแถวนั้น
Edge — ลบ Sale ต้อง bulk reassign ลูกค้าก่อน (ปอนด์ยืนยัน) Given ผู้ใช้ที่เป็น Sale ดูแลลูกค้า 12 ราย และมี PO ค้าง When Admin กด "ลบผู้ใช้" Then ระบบเปิดหน้าจอ bulk reassign บังคับเลือก Sale ปลายทางรับช่วงลูกค้าทั้ง 12 ราย; ต้องเลือกให้ครบก่อน จึงจะกด mark ลบได้; PO เดิมเดินต่อ, ห้ามสร้าง PO ใหม่ในนามผู้ถูกลบ — ดู Deletion Sale
Error — ยืนยันลบโดยยังไม่ reassign / ไม่มีสิทธิ์ D Given ยังไม่เลือกผู้รับช่วงครบ หรือผู้ใช้ปัจจุบันไม่มีสิทธิ์ Delete/Admin ของ Settings When กดยืนยันลบ Then ไม่ลบ + error "ต้องมอบหมายลูกค้าทั้งหมดให้ผู้รับช่วงก่อน" หรือ "ไม่มีสิทธิ์ลบผู้ใช้"

US-SET-03 — หน้าจอ 3: Config VAT + effective date + ประวัติ Must

As a Admin, I want ตั้งอัตรา VAT พร้อมวันที่มีผลและเก็บประวัติ, so that ใบกำกับภาษีคิดอัตราถูกตามช่วงเวลาที่ออกใบ

Happy Given แท็บ Config VAT ปัจจุบัน 7% When เพิ่มรายการ VAT 7% effective 01/01/2569 แล้วบันทึก Then บันทึก + เก็บประวัติเป็นรายการ (อัตรา/วันมีผล/ผู้ตั้ง); ใบกำกับภาษียึดอัตราตาม invoice date (ดู US-INV-02)
Edge — หลายรายการตามช่วงเวลา Given มี VAT 7% (มีผล 2565) และ 7% (มีผล 2569) When ออกใบกำกับลงวันที่ในปี 2568 Then ระบบเลือกอัตราที่ effective ครอบวันที่นั้น (รายการปี 2565); การเปลี่ยน VAT ใหม่ไม่กระทบใบเก่า
Error — effective date ว่าง/ทับซ้อน Given effective date เว้นว่าง หรือซ้ำวันที่มีผลกับรายการเดิม When บันทึก Then error "ต้องระบุวันที่มีผลที่ไม่ทับซ้อนกับรายการเดิม" — ไม่บันทึก

US-SET-04 — หน้าจอ 4: ข้อมูลบริษัท (ใช้บนใบกำกับ) Should

As a Admin, I want กรอกข้อมูลบริษัทที่ต้องปรากฏบนใบกำกับภาษี, so that เอกสารออกถูกต้องตามสรรพากรและมีโลโก้บริษัท

Happy Given แท็บข้อมูลบริษัท When กรอกชื่อบริษัท / เลขผู้เสียภาษี 13 หลัก / ที่อยู่ / เบอร์ / อีเมล และอัปโหลด logo แล้วบันทึก Then บันทึก; ข้อมูลชุดนี้ปรากฏบน invoice-print (หัวใบกำกับภาษี) — ดู Invoice
Edge — logo placeholder Given ยังไม่อัปโหลด logo When ดูใบกำกับ Then แสดง placeholder logo (ตัว "E"/กล่องว่าง) แทน — ไม่ error, ยังพิมพ์ใบได้
Error — เลขผู้เสียภาษีผิดรูปแบบ Given กรอกเลขผู้เสียภาษีไม่ครบ 13 หลัก หรือมีตัวอักษร When บันทึก Then error "เลขประจำตัวผู้เสียภาษีต้องเป็นตัวเลข 13 หลัก" — ไม่บันทึก

US-SET-05 — หน้าจอ 5: Audit log (คอลัมน์/filter/search + สัมพันธ์ trace) Should

As a Admin/ผู้ตรวจสอบ, I want ดู audit log การเปลี่ยนแปลงทั้งระบบพร้อมคอลัมน์และตัวกรอง, so that ตรวจสอบได้ว่าใครทำอะไรเมื่อไหร่โดยไม่ต้องไปหลายหน้า

Happy — คอลัมน์ + filter + search Given แท็บ Audit log When กรองด้วย ผู้ใช้ = "สมหญิง" + module = "ลูกค้า" + ช่วงวันที่ + พิมพ์คำค้น "Follow-up" Then แสดงตารางคอลัมน์ (เวลา / ผู้ทำ / module / entity / field / จาก→เป็น / เหตุผล) + pagination; sort ได้ตามเวลา (ใหม่→เก่า default)
Edge — ความสัมพันธ์กับ Traceability Given audit log ใน Settings เป็น "มุมมองรวมทั้งระบบ" ของ field-level audit เดียวกับ Traceability When คลิกแถว audit Then deep link ไปหน้า trace ของ entity นั้น (แหล่งข้อมูลเดียวกัน ไม่ซ้ำซ้อน); retention online 1 ปี, purge/archive = Super User
Error — ไม่มีสิทธิ์ดู audit Given ผู้ใช้ไม่มีสิทธิ์ Read Settings/Admin When พยายามเปิดแท็บ Audit log หรือ URL ตรง Then guard 403 / ไม่เห็นแท็บ (audit = ข้อมูลไวต่อความปลอดภัย)
