จุดที่อยากให้ BA ตรวจเข้ม (cross-review journeys ↔ stories)
1. เจ้าของ auto-transition ข้าม module: "ยืนยัน PO ใบแรก → Lead→Active" (①/②) — เป็นของ US-PO-02 หรือ US-CUS-02? อย่าให้ตกหล่นหรือซ้ำ 2 เจ้า · เช่นเดียวกับ "ทุก PRD ผ่าน → PO พร้อมจัดส่ง" (③/④) = cascade ที่ US-QC-01 ทำ — ยืนยัน PO module ไม่ได้ประกาศ transition นี้ซ้ำ
2. Negative stock มี 2 จังหวะเตือน: ③ เตือนทั้งตอน "รับงาน" (US-PRD-01 edge) และตอน "เริ่มผลิต" (US-PRD-02 edge) — BA ยืนยันว่ามี AC ครบทั้ง 2 จุด ไม่ใช่จุดเดียว
3. Rework loop ต่อเนื่อง: ③↔④ (ตก→Rework→ผลิตซ้ำ→run ถัดไป→ตรวจซ้ำเห็นประวัติ) — ไล่ว่า US-PRD-04 + US-QC-02 ครอบทุก step ของวงรอบ (badge/feedback/run history) ไม่ขาดตอน
4. DN ปฏิเสธ → raise Sale → อาจ Follow-up: ⑤→① ข้าม 3 module (US-SHP-02 → US-PO-05/US-CUS-03) — ยืนยัน chain นี้มี story รับช่วงครบ ไม่มีรูตรงกลาง
5. Home task-inbox = ปลายทาง noti ทุกสาย: ทุก 🔔 ในทั้ง 6 สาย ต้องโผล่ใน Home ของ role ปลายทาง (US-HOME-01) และนับตรงกับ dashboard tile — BA ตรวจว่าเหตุการณ์ครบ
6. Stock Reservation (r5): จอง (Confirmed, US-PO-02) → ตัดจริง (เริ่มผลิต, US-PRD-02) → คืน/dialog (cancel, US-PO-04) → ปรับ delta (Hold edit, US-PO-05) — BA ยืนยันทุก step ของ reservation lifecycle มี AC รองรับ + 3 ยอด (US-STK-03) + ใครจอง (US-STK-06) ครบ ไม่มีกติกาใหม่ (อ้าง stock-reservation.md)
7. ไม่มีกติกาใหม่: ทุก step อ้าง entity-status-map cascade C1–C20 — ถ้า journey เล่าอะไรที่ story/AC ไม่มี = ต้องเพิ่ม AC (ไม่ใช่เพิ่มกติกา)
