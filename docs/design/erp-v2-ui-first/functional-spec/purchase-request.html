3. User Stories + Acceptance Criteria

US-PR-01 — PR auto จาก PO วัตถุดิบขาด Must

As a ระบบ, I want สร้าง PR อัตโนมัติเมื่อ PO มีวัตถุดิบขาด (เฉพาะส่วนที่ขาด), so that ฝ่ายคลังจัดซื้อทันโดย PO ไม่ถูกบล็อก

Happy Given PO-185 ต้องใช้กลีเซอรีนเกินยอดคงเหลือ 6 กก. When ยืนยัน PO Then gen PR-000031 (กลีเซอรีน 6 กก. = ส่วนที่ขาด) ผูก PO-185; โผล่ Stock + Production Dashboard; ยิง Noti (C15)
Edge — มี PR ค้างของวัตถุดิบเดิม Given มี PR เปิดค้างของกลีเซอรีนอยู่แล้ว When PO ใหม่ขาดกลีเซอรีนอีก Then สร้าง PR ใบใหม่แยก (ตามคำตอบปอนด์: สร้างใบใหม่ทุกครั้ง ไม่รวมใบเดิม)
Error — วัตถุดิบพอ Given วัตถุดิบพอทุกตัว When ยืนยัน PO Then ไม่สร้าง PR

US-PR-02 — สร้าง PR ตรง + รับทราบ/ปิด manual Should

As a ฝ่ายคลัง, I want สร้าง PR เองและปรับสถานะ รับทราบ/ปิดคำขอ, so that จัดซื้อเชิงรุกได้

Happy Given ฝ่ายคลังอยู่หน้า pr-create When สร้าง PR-000028 แอลกอฮอล์ 250 ล. Then PR=เปิดคำขอ; กด "รับทราบ" → Acknowledged
Edge — ยกเลิก Given PR ที่ยังไม่รับของ When กด "ยกเลิก" + comment Then PR=ยกเลิก (comment บังคับ) + trace
Error — ยกเลิกไม่ใส่ comment Given comment ว่าง When ยกเลิก Then error "ต้องระบุเหตุผล"

US-PR-03 — Fulfilled/Partially auto จาก GR Must

As a ระบบ, I want อัปเดตสถานะ PR อัตโนมัติจากการรับเข้า (GR), so that สถานะสะท้อนของจริง

Happy Given PR-000031 (6 กก.) มี GR รับครบ 6 กก. When บันทึก GR ผูก PR Then PR-000031=ของเข้าครบ (Fulfilled) auto + ระบุ lot (C17)
Edge — รับบางส่วน Given PR-000028 (250 ล.) รับ 200 ล. When GR บันทึก Then PR-000028=รับบางส่วน + เสนอสร้าง PR-000032 (50 ล.) รอ user review
Error — GR อ้าง PR ผิดวัตถุดิบ Given GR line วัตถุดิบไม่ตรงกับ PR ที่อ้าง When ยืนยัน Then error "วัตถุดิบไม่ตรงกับคำขอ" — ไม่ปิด PR
