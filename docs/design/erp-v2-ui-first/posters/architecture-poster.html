07Database

🗄️ MySQL 8 (InnoDB)

System of record · Cloud SQL in Phase 3

Holds the append-only stock ledger (negative stock allowed, FIFO retro-link), field-level audit log (1-year online), gapless sequence counters, reservations, and all business tables. Daily backup.

MySQL 8 / InnoDBstock ledgeraudit_logsequencereservation
⚙️ Runs alongside the backend — platform capabilities wired into every layer

💾 Storage Abstraction ADR-006

Single StorageService for attachments/archives. Local disk (Phase 2) → Google Cloud Storage (Phase 3) by config — business code unchanged.

⏰ Scheduled Jobs J1–J7

Session reset 06:00 · inactivity sweep · invoice overdue · potential-delay · outbox dispatcher · nightly stock/ledger integrity · DB backup. All idempotent, tracked in scheduled_job_run.

🔔 Notification Outbox ADR-005

Events written in the same transaction as the status change (never lost). Dispatcher fans out to everyone with the Read bit; deep-link + per-user acknowledge.

🧭 One codebase, environment by config

Business logic never depends on where it runs. From factory PC (Phase 2) to Google Cloud (Phase 3), only environment configuration changes — the storage driver, database endpoint and OAuth credentials swap out; services, ORM and schema stay identical (ADR-006). Every layer above is present and required; none is optional.

Browser → React → HTTPS → Gateway → Auth/RBAC → NestJS → Prisma → MySQL + Storage · Scheduled Jobs · Notification Outbox
